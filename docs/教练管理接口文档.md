# 教练管理

## 通用说明

### 统一响应格式

```json
{
  "state": 20000,       // 业务状态码
  "message": null,      // 错误时携带提示信息
  "data": { ... }       // 成功时携带数据
}
```

| state | 含义 |
|-------|------|
| 20000 | 成功 |
| 40000 | 请求参数错误 |
| 40100 | 未登录或 token 过期 |
| 40300 | 权限不足 |
| 40400 | 资源不存在 |
| 40900 | 数据冲突（如用户名已存在） |

### 认证方式

所有接口（除登录和注册外）需要在请求头中携带 Token：

```
Authorization: Bearer <token>
```

### 角色说明

| 值 | 角色 |
|----|------|
| 1 | 学员 |
| 2 | 教练 |
| 3 | 管理员 |

### 分页接口通用返回

分页接口返回 `Page<T>` 对象：

```json
{
  "state": 20000,
  "data": {
    "records": [...],   // 当前页数据
    "total": 56,        // 总记录数
    "pages": 6,         // 总页数
    "current": 1,       // 当前页码
    "size": 10          // 每页条数
  }
}
```

基础路径：`/coaches`

### 4.1 分页查询教练

```
GET /coaches?page=1&size=10&username=coach&realName=张
```

| 参数 | 说明 | 必填 |
|------|------|------|
| page | 页码，默认 1 | 否 |
| size | 每页条数，默认 10 | 否 |
| username | 用户名模糊搜索（LIKE %keyword%） | 否 |
| realName | 姓名模糊搜索（LIKE %keyword%） | 否 |
| vehicleType | 准驾车型精确匹配（如 C1/C2，使用 FIND_IN_SET 匹配逗号分隔字段） | 否 |

> 任意登录用户可调用，支持多条件组合查询。
> - `?realName=张` 搜姓张的教练
> - `?username=coach` 搜用户名为 coach 开头的
> - `?vehicleType=C1` 搜准教车型包含 C1 的教练
> - 以上参数可任意组合使用

**响应（包含关联的用户信息）：**
```json
{
  "state": 20000,
  "data": {
    "records": [
      {
        "coachId": 1,
        "userId": 2,
        "rating": 4.8,
        "coachYears": 6,
        "vehicleType": "C1",
        "availableTime": null,
        "realName": "张教练",
        "username": "coach1",
        "phone": "13812340001"
      },
      {
        "coachId": 2,
        "userId": 3,
        "rating": 4.5,
        "coachYears": 10,
        "vehicleType": "C1,C2",
        "availableTime": null,
        "realName": "李教练",
        "username": "coach2",
        "phone": "13812340002"
      }
    ],
    "total": 2,
    "pages": 1,
    "current": 1,
    "size": 10
  }
}
```

> 返回的数据包含教练扩展信息和关联的用户信息（realName/username/phone），前端可直接展示。

    "idCard": "510101199505012345",

### 4.3 新增教练

```
POST /coaches
Content-Type: application/json

{
  "userId": 2,
  "rating": 4.8,
  "coachYears": 6,
  "vehicleType": "C1,C2,B1"
}
```

> - 仅管理员可调用
> - `vehicleType` 以逗号分隔（如 `"C1,C2"`），用于教练推荐匹配

### 4.4 修改教练

```
PUT /coaches/{coachId}
Content-Type: application/json

{
  "rating": 4.9,
  "coachYears": 7,
  "vehicleType": "C1,C2,B1"
}
```

> 仅管理员可调用

### 4.5 删除教练

```
DELETE /coaches/{coachId}
```

> 仅管理员可调用，逻辑删除

### 6.1 学员申请教练

```
POST /coach-applications
Content-Type: application/x-www-form-urlencoded

studentId=4&coachId=1
```

> 仅学员可调用，提交后等待管理员审核

### 6.2 查询待审核的教练申请

```
GET /coach-applications/pending
```

> 仅管理员可调用

### 6.3 查询学员的申请记录

```
GET /coach-applications/student/{studentId}
```

> 任意登录用户可调用，学员可查看自己的申请记录和审核状态

### 6.4 审核教练申请

```
PUT /coach-applications/{id}/audit?pass=true
```

| 参数 | 说明 | 必填 |
|------|------|------|
| pass | true=通过，false=拒绝 | 是 |
| reason | 拒绝原因（pass=false 时选填） | 否 |

> - 仅管理员可调用
> - 审核通过时会自动写入 `student_coach` 绑定关系

### 6.5 自动推荐教练

```
GET /coach-assignments/recommend?studentId=4&topN=5
```

> 任意登录用户可调用
> 推荐算法：根据学员的 `licenseType` 匹配教练的 `vehicleType`（FIND_IN_SET）→ 按评分降序

### 6.6 手动分配教练

```
POST /coach-assignments
Content-Type: application/x-www-form-urlencoded

studentId=4&coachId=2
```

> 仅管理员可调用。若学员已绑定教练则会报错（需先解绑）

### 6.7 查询所有绑定关系

```
GET /coach-assignments
```

> 仅管理员可调用。返回包括学员姓名和教练姓名：

```json
{
  "state": 20000,
  "data": [
    {
      "id": 1,
      "studentId": 4,
      "coachId": 1,
      "bindTime": "2026-05-22T12:00:00",
      "studentName": "王小明",
      "coachName": "张教练"
    }
  ]
}
```

### 6.8 解绑教练

```
PUT /coach-assignments/{id}/unbind
```

> 仅管理员可调用，将 `student_coach.status` 置为 0（已解绑）
