# 教练移交学员接口文档

## 通用说明

### 统一响应格式

```json
{
  "state": 20000,
  "message": null,
  "data": { ... }
}
```

### 业务状态码

| state | 含义 |
|-------|------|
| 20000 | 成功 |
| 40000 | 请求参数错误 |
| 40100 | 未登录或 token 过期 |
| 40300 | 权限不足 |
| 40400 | 资源不存在 |
| 40900 | 数据冲突（如已有待审核移交申请） |

### 认证方式

所有接口需在请求头中携带 Token：

```
Authorization: Bearer <token>
```

### 角色说明

| 值 | 角色 |
|----|------|
| 1 | 学员 |
| 2 | 教练 |
| 3 | 管理员 |

### 申请状态说明

| status | 含义 |
|--------|------|
| 0 | 待审核 |
| 1 | 已通过（已自动完成移交） |
| 2 | 已拒绝 |

## 一、教练端接口

基础路径：`/coach-portal`

### 1.1 教练申请移交学员

教练因故（如生病、离职等）无法继续带教学员，发起将名下某学员移交给另一位指定教练的申请，由管理员审核。

```
POST /coach-portal/student-transfers?studentId=5&targetCoachId=1&reason=因身体原因无法继续带教
```

#### 请求参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| studentId | int | 是 | 要移交的学员 user_id |
| targetCoachId | int | 是 | 接收学员的教练 coach_id |
| reason | String | 是 | 移交原因（如"因身体原因无法继续带教"） |

#### 校验规则

| 场景 | 行为 |
|------|------|
| 该学员不是当前教练名下的学员 | 返回 40000 "该学员不是您名下的学员" |
| 该学员已有同一教练发起的待审核移交 | 返回 40900 "该学员的移交申请已提交，请等待管理员处理" |

#### 请求示例

```
POST /coach-portal/student-transfers?studentId=5&targetCoachId=1&reason=因身体原因无法继续带教
```

#### 成功响应

```json
{
  "state": 20000,
  "data": null
}
```

#### 错误示例

**400 — 学员不在名下：**
```json
{
  "state": 40000,
  "message": "该学员不是您名下的学员",
  "data": null
}
```

**409 — 已有待审核的移交申请：**
```json
{
  "state": 40900,
  "message": "该学员的移交申请已提交，请等待管理员处理",
  "data": null
}
```

---

### 1.2 查看本人的移交申请记录

教练查看自己发起过的所有学员移交申请历史，按申请时间倒序排列。

```
GET /coach-portal/student-transfers
```

#### 请求参数

无

#### 成功响应

```json
{
  "state": 20000,
  "data": [
    {
      "id": 2,
      "studentId": 5,
      "targetCoachId": 1,
      "transferReason": "因身体原因无法继续带教",
      "status": 0,
      "auditReason": null,
      "applyTime": "2026-06-05T11:00:00",
      "auditTime": null,
      "studentName": "李芳"
    },
    {
      "id": 3,
      "studentId": 6,
      "targetCoachId": 2,
      "transferReason": "学员需更换至对应车型教练",
      "status": 1,
      "auditReason": null,
      "applyTime": "2026-06-04T09:00:00",
      "auditTime": "2026-06-04T10:00:00",
      "studentName": "赵强"
    }
  ]
}
```

#### 返回字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| id | int | 申请ID |
| studentId | int | 学员 user_id |
| targetCoachId | int | 目标接收教练 coach_id |
| transferReason | String | 移交原因 |
| status | int | 0-待审核, 1-已通过, 2-已拒绝 |
| auditReason | String | 拒绝原因（通过或待审核时为 null） |
| applyTime | datetime | 申请时间 |
| auditTime | datetime | 审核时间（待审核时为 null） |
| studentName | String | 学员真实姓名 |

#### 前端展示建议

根据 `status` 显示不同状态的标签：

| status | 显示 |
|--------|------|
| 0 | 待审核（灰色/黄色标签） |
| 1 | 已通过（绿色标签，提示"已移交"） |
| 2 | 已拒绝（红色标签，显示 `auditReason`） |

## 二、管理员端接口

教练移交申请的审核与学员申请共用同一套管理员接口，通过 `applyType` 字段区分。

### 2.1 查询待审核列表（含移交申请）

管理员查看所有待审核申请，其中 `applyType="教练移交"` 的记录即为教练移交申请。

```
GET /coach-applications/pending
```

完整响应参考《教练申请审核接口文档》。涉及移交的记录包含以下额外信息：

| 字段 | 类型 | 说明 |
|------|------|------|
| sourceCoachName | String | 发起移交的教练姓名 |
| transferReason | String | 移交原因 |
| applyType | String | "教练移交" |

---

### 2.2 审核移交申请

```
PUT /coach-applications/{id}/audit?pass=true
```

同《教练申请审核接口文档》2.2 节。

**通过时的额外校验**：
- 检查发起移交的教练是否仍与学员有绑定关系（防止中途已被其他操作解绑）
- 如已无绑定，返回 40900 "该学员已不是发起移交的教练名下的学员，请核实"

**通过时的后台操作**：
1. 解绑旧教练（将原绑定关系置为 `status=0`）
2. 绑定新教练（创建 `student_coach` 记录，`coach_id = targetCoachId`）

## 三、业务流程

### 教练移交学员完整流程

```
教练端                           管理员端
  │                                │
  ├─ 查看名下学员列表               │
  │  GET /coach-portal/students    │
  │  → 确认要移交的学员             │
  │                                │
  ├─ 提交移交申请                   │
  │  POST /coach-portal/           │
  │  student-transfers             │
  │  ?studentId=5&targetCoachId=1  │
  │  &reason=因身体原因无法继续带教  │
  │                                │
  │                    ┌────────────┤
  │                    │ 查询待审核列表 │
  │                    │ GET /coach-  │
  │                    │ applications/ │
  │                    │ pending      │
  │                    │ → applyType  │
  │                    │   ="教练移交" │
  │                    ├────────────┤
  │                    │ 审核移交申请  │
  │                    │ PUT /coach-  │
  │                    │ applications/ │
  │                    │ {id}/audit   │
  │                    │ ?pass=true   │
  │                    │             │
  │                    │ 通过时自动：  │
  │                    │ 解绑旧教练 →  │
  │                    │ 绑定新教练    │
  │                    └──────┬─────┘
  │                           │
  ├─ 查看申请结果               │
  │  GET /coach-portal/       │
  │  student-transfers        │
  │  → 看到 status 变为 1 或 2 │
  │                           │
  │  如已通过：学员已不在名下     │
  │  GET /coach-portal/        │
  │  students → 列表中已移除    │
```

## 四、接口路径速查

### 教练端

| 操作 | 方法 | 路径 | 角色 |
|------|------|------|------|
| 查看名下学员列表 | GET | `/coach-portal/students` | 教练 |
| 申请移交学员 | POST | `/coach-portal/student-transfers` | 教练 |
| 查看本人移交记录 | GET | `/coach-portal/student-transfers` | 教练 |

### 管理员端（共用教练申请审核接口）

| 操作 | 方法 | 路径 | 角色 |
|------|------|------|------|

| 审核申请（含移交） | PUT | `/coach-applications/{id}/audit` | 管理员 |

## 五、前端对接建议

### 教练端页面

**学员列表页（触发移交入口）：**
- 调 `GET /coach-portal/students` 展示名下学员
- 每个学员卡片/行增加"移交"按钮
- 点击后弹窗：选择目标教练 + 填写移交原因
- 确认 → 调 `POST /coach-portal/student-transfers`
- 成功提示"移交申请已提交，等待管理员审核"

**移交记录页：**
- 调 `GET /coach-portal/student-transfers` 展示列表
- 用 `status` 字段区分显示状态标签
- 已拒绝的显示 `auditReason`

### 管理员端页面

**待审核列表页：**
- 调 `GET /coach-applications/pending` 展示
- 通过 `applyType="教练移交"` 筛选或标记移交申请
- 卡片展示格式：`源教练 → 学员 → 目标教练` + 移交原因
- 提供"通过"和"拒绝"操作按钮

**审核操作：**
- 通过 → 调 `PUT /coach-applications/{id}/audit?pass=true`
- 拒绝 → 弹窗填写原因 → 调 `PUT /coach-applications/{id}/audit?pass=false&reason=原因`
- 通过后自动解绑旧教练并绑定新教练，无需额外操作
