# 教练分配管理接口文档

## 通用说明

### 统一响应格式

```json
{
  "state": 20000,
  "message": null,
  "data": { ... }
}
```

### 业务状态码

| state | 含义 |
|-------|------|
| 20000 | 成功 |
| 40000 | 请求参数错误 |
| 40100 | 未登录或 token 过期 |
| 40300 | 权限不足 |
| 40400 | 资源不存在 |
| 40900 | 数据冲突（如学员已绑定教练） |

### 认证方式

所有接口需在请求头中携带 Token：

```
Authorization: Bearer <token>
```

### 角色说明

| 值 | 角色 |
|----|------|
| 1 | 学员 |
| 2 | 教练 |
| 3 | 管理员 |

## 一、管理员端接口

基础路径：`/coach-assignments`

### 1.1 查询所有绑定关系

管理员查看当前所有正常绑定的学员-教练关系列表。

```
GET /coach-assignments
```

#### 请求参数

无

#### 成功响应

```json
{
  "state": 20000,
  "data": [
    {
      "id": 1,
      "studentId": 4,
      "coachId": 1,
      "bindTime": "2026-06-04T12:00:00",
      "studentName": "王小明",
      "coachName": "张教练"
    },
    {
      "id": 2,
      "studentId": 5,
      "coachId": 2,
      "bindTime": "2026-06-04T12:00:00",
      "studentName": "李芳",
      "coachName": "李教练"
    }
  ]
}
```

#### 返回字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| id | int | 绑定记录ID |
| studentId | int | 学员 user_id |
| coachId | int | 教练 coach_id |
| bindTime | datetime | 绑定时间 |
| studentName | String | 学员真实姓名 |
| coachName | String | 教练真实姓名 |

---

### 1.2 按教练查询名下学员

管理员指定一个教练，查看其名下的学员列表。

```
GET /coach-assignments/coach/{coachId}/students
```

#### 请求参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| coachId | int | 路径参数 | 教练 coach_id |

#### 成功响应

```json
{
  "state": 20000,
  "data": {
    "coachId": 1,
    "coachName": "张教练",
    "studentCount": 2,
    "students": [
      {
        "bindId": 1,
        "studentId": 4,
        "bindTime": "2026-06-04T12:00:00",
        "realName": "王小明",
        "phone": "15912340001",
        "licenseType": "C1",
        "idCard": "500101200001011234"
      },
      {
        "bindId": 3,
        "studentId": 6,
        "bindTime": "2026-06-04T14:00:00",
        "realName": "赵强",
        "phone": "15912340003",
        "licenseType": "C1",
        "idCard": "500101199901013456"
      }
    ]
  }
}
```

#### 返回字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| coachId | int | 教练ID |
| coachName | String | 教练真实姓名 |
| studentCount | int | 学员总数 |
| students | Array | 学员列表 |
| ∟ bindId | int | 绑定记录ID |
| ∟ studentId | int | 学员 user_id |
| ∟ bindTime | datetime | 绑定时间 |
| ∟ realName | String | 学员真实姓名 |
| ∟ phone | String | 学员手机号 |
| ∟ licenseType | String | 学员报考车型 |
| ∟ idCard | String | 学员身份证号 |

#### 错误示例

**404 — 教练不存在：**
```json
{
  "state": 40400,
  "message": "教练不存在",
  "data": null
}
```

### 1.3 自动推荐教练

管理员为学员分配教练之前，根据学员报考车型匹配准教车型，按评分排序返回推荐列表。

```
GET /coach-assignments/recommend?studentId=4&topN=5
```

#### 请求参数

| 参数 | 类型 | 必填 | 默认 | 说明 |
|------|------|------|------|------|
| studentId | int | 是 | — | 学员 user_id |
| topN | int | 否 | 5 | 返回前 N 条推荐结果 |

#### 成功响应

```json
{
  "state": 20000,
  "data": [
    {
      "coachId": 1,
      "userId": 2,
      "rating": 4.8,
      "coachYears": 6,
      "vehicleType": "C1",
      "availableTime": null
    },
    {
      "coachId": 2,
      "userId": 3,
      "rating": 4.5,
      "coachYears": 10,
      "vehicleType": "C1,C2",
      "availableTime": null
    }
  ]
}
```

#### 错误示例

```json
{
  "state": 40400,
  "message": "学员不存在或未填写报考车型",
  "data": null
}
```

### 1.4 手动分配教练

管理员直接为学员绑定一个教练。学员当前不能有激活的绑定关系，如有需先解绑。

```
POST /coach-assignments?studentId=4&coachId=1
```

#### 请求参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| studentId | int | 是 | 学员 user_id |
| coachId | int | 是 | 教练 coach_id |

#### 成功响应

```json
{
  "state": 20000,
  "data": null
}
```

#### 校验规则

| 场景 | 行为 |
|------|------|
| 学员已绑定其他教练 | 返回 40900 "该学员已绑定教练，请先解绑" |

---

### 1.5 解绑教练

管理员将某条绑定关系置为无效（`status=0`）。解绑后该学员变为"无教练"状态。

```
PUT /coach-assignments/{id}/unbind
```

#### 请求参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | int | 路径参数 | student_coach 表绑定记录 ID |

#### 成功响应

```json
{
  "state": 20000,
  "data": null
}
```

#### 错误示例

```json
{
  "state": 40400,
  "message": "绑定记录不存在",
  "data": null
}
```

## 二、学员端接口

基础路径：`/coach-assignments`

### 2.1 查询我的教练

学员查询自己当前绑定的教练详细信息。

```
GET /coach-assignments/my-coach/{studentId}
```

#### 请求参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| studentId | int | 路径参数 | 学员 user_id |

#### 成功响应（已绑定）

```json
{
  "state": 20000,
  "data": {
    "coachId": 1,
    "userId": 2,
    "realName": "张教练",
    "phone": "13812340001",
    "rating": 4.8,
    "coachYears": 6,
    "vehicleType": "C1",
    "availableTime": "{\"monday\":[\"09:00-12:00\"]}",
    "bindTime": "2026-06-04T12:00:00"
  }
}
```

#### 成功响应（未绑定）

```json
{
  "state": 20000,
  "data": null
}
```

#### 返回字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| coachId | int | 教练ID |
| userId | int | 教练关联的用户ID |
| realName | String | 教练真实姓名 |
| phone | String | 教练手机号 |
| rating | decimal | 综合评分 (1.0-5.0) |
| coachYears | int | 执教年限 |
| vehicleType | String | 准教车型，逗号分隔 |
| availableTime | String | 空闲时间 JSON |
| bindTime | datetime | 绑定时间 |

## 三、业务流程

### 管理员分配教练完整流程

```
管理员端
  │
  ├─ 1. 查询目标学员是否已有教练
  │    GET /coach-assignments/my-coach/{studentId}
  │    → 如 data!=null 则已有教练
  │
  ├─ 2. 如有教练，先解绑
  │    GET /coach-assignments → 找到绑定记录 ID
  │    PUT /coach-assignments/{id}/unbind
  │
  ├─ 3. 获取推荐教练列表（可选）
  │    GET /coach-assignments/recommend?studentId=4&topN=5
  │
  ├─ 4. 手动分配
  │    POST /coach-assignments?studentId=4&coachId=1
  │
  └─ 5. 验证分配结果
       GET /coach-assignments/my-coach/{studentId}
       → 返回教练信息
```

## 四、接口路径速查

| 操作 | 方法 | 路径 | 角色 |
|------|------|------|------|
| 查询所有绑定关系 | GET | `/coach-assignments` | 管理员 |
| 按教练查学员 | GET | `/coach-assignments/coach/{coachId}/students` | 管理员 |
| 自动推荐教练 | GET | `/coach-assignments/recommend` | 学员/管理员 |
| 手动分配教练 | POST | `/coach-assignments` | 管理员 |
| 解绑教练 | PUT | `/coach-assignments/{id}/unbind` | 管理员 |
| 查询我的教练 | GET | `/coach-assignments/my-coach/{studentId}` | 全部登录 |

## 五、前端对接建议

### 管理员端页面

**教练分配页：**
1. 搜索学员 → 调 `GET /students` 分页搜索学员
2. 查看学员当前教练 → 调 `GET /coach-assignments/my-coach/{studentId}`
3. 如有教练，显示"当前已绑定：XXX"，提供"解绑"按钮
4. 推荐教练列表 → 调 `GET /coach-assignments/recommend?studentId=4` 展示
5. 手动选择教练并确认分配 → 调 `POST /coach-assignments`

**教练详情查看页：**
1. 搜索教练 → 调其他教练管理接口
2. 查看教练名下学员 → 调 `GET /coach-assignments/coach/{coachId}/students`
3. 展示学员列表，提供单个学员的解绑/重新分配入口

**绑定总览页：**
- 调 `GET /coach-assignments` 展示全部绑定关系
- 每条记录后面提供"解绑"操作入口
