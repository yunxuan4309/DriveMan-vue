# 分页与搜索功能增强 — 接口变更说明（第二批）

> 日期：2026-06-10

针对费用标准、车型配置、特种车辆考试、准教车型审核、合场管理、支付管理、二次培训、体检管理 8 个功能模块新增分页与多条件搜索。

**⚠️ 重要：以下所有列表接口均已改为分页返回，前端必须实现分页组件（页码 + 每页条数），否则无法展示数据。**

---

## ① 费用标准 — 分页 + 车型搜索

### 接口

```http
GET /fee-standards
```

### 变更说明

原接口返回全量列表，现改为分页返回。

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| page | int | 否 | 页码，默认 1 |
| size | int | 否 | 每页条数，默认 10 |
| licenseType | String | 否 | 车型筛选（C1/C2/...） |

### 响应

```json
{
  "state": 200,
  "data": {
    "records": [ ... ],
    "total": 50,
    "current": 1,
    "size": 10,
    "pages": 5
  }
}
```

其余接口（`POST /fee-standards`、`PUT /fee-standards/{id}`、`DELETE /fee-standards/{id}`）不变。

## ③ 特种车辆考试 — 新建 Controller + 分页搜索

之前无 Controller，现完整新增以下接口。

### 分页查询（含多条件搜索）

```http
GET /special-exam-records
```

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| page | int | 否 | 页码，默认 1 |
| size | int | 否 | 每页条数，默认 10 |
| studentName | String | 否 | 学员姓名关键词 |
| licenseType | String | 否 | 车型筛选（N1/N2/N3） |
| subject | int | 否 | 科目：1-理论, 2-实操 |
| passStatus | int | 否 | 是否合格：0-不合格, 1-合格 |

### 其余接口

```http
GET    /special-exam-records/{id}          # 查询单条
POST   /special-exam-records               # 新增
PUT    /special-exam-records/{id}          # 修改
DELETE /special-exam-records/{id}          # 删除
PUT    /special-exam-records/{id}/score    # 录入成绩（含合格状态/补考次数/证书编号）
```

## 前端对接注意事项

1. **所有 `GET` 列表接口已改为分页返回**，响应体为 `Page` 结构（`records`/`total`/`current`/`size`/`pages`），不再是原始数组。
2. **每页默认 10 条**，前端需提供页码和每页条数控制组件。
3. **搜索参数均为可选**，不传时返回全部数据（分页）。
4. 部分原全量列表接口保留（如 `GET /familiarizations/all`、`GET /payment-records/my`），仅影响管理端接口。
