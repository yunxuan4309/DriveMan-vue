# 教练准教车型变更申请接口文档

## 通用说明

### 统一响应格式

```json
{
  "state": 20000,
  "message": null,
  "data": { ... }
}
```

### 业务状态码

| state | 含义 |
|-------|------|
| 20000 | 成功 |
| 40000 | 请求参数错误 |
| 40100 | 未登录或 token 过期 |
| 40300 | 权限不足 |
| 40400 | 资源不存在 |
| 40900 | 数据冲突（如已有待审核申请） |

### 认证方式

所有接口需在请求头中携带 Token：

```
Authorization: Bearer <token>
```

### 角色说明

| 值 | 角色 |
|----|------|
| 1 | 学员 |
| 2 | 教练 |
| 3 | 管理员 |

### 申请状态说明

| status | 含义 |
|--------|------|
| 0 | 待审核 |
| 1 | 已通过（已自动更新教练准教车型） |
| 2 | 已拒绝 |

## 一、教练端接口

基础路径：`/coach-portal`

### 1.1 提交准教车型变更申请

教练提交增加/变更可教车型的申请，提交后由管理员审核。

```
POST /coach-portal/vehicle-applications
```

#### 请求参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| requestedVehicleType | String | 是 | 申请的新准教车型，多个车型用英文逗号分隔，如 "C1,C2" |
| applyReason | String | 否 | 申请理由（如 "已取得 C2 教练资格证"） |

#### 校验规则

1. **教练信息必须存在** — 当前登录用户必须已绑定教练信息
2. **车型不能为空** — `requestedVehicleType` 不能为空或空白
3. **车型必须在系统中存在** — 每个车型必须存在于 `license_config` 表的 `license_type` 列（当前系统支持的车型：C1、C2、B1、N1、N2、N3）
4. **不能与当前车型相同** — 新车型不能与当前 `coach.vehicle_type` 完全一致
5. **不可重复提交** — 同一教练不能有状态为"待审核"的申请存在

#### 请求示例

```
POST /coach-portal/vehicle-applications?requestedVehicleType=C1,C2&applyReason=已取得C2教练资格证
```

#### 成功响应（20000）

```json
{
  "state": 20000,
  "data": null
}
```

#### 错误示例

**409 — 已有待审核申请：**
```json
{
  "state": 40900,
  "message": "您已有一条待审核的申请，请等待处理",
  "data": null
}
```

**400 — 车型不存在：**
```json
{
  "state": 40000,
  "message": "准教车型 N9 不存在，系统支持的车型为: C1, C2, B1, N1, N2, N3",
  "data": null
}
```

**400 — 与当前车型一致：**
```json
{
  "state": 40000,
  "message": "申请车型与当前准教车型一致，无需变更",
  "data": null
}
```

### 1.2 查看本人的申请记录

教练查看自己提交过的所有申请记录，按申请时间倒序排列。

```
GET /coach-portal/vehicle-applications
```

#### 请求参数

无

#### 成功响应

```json
{
  "state": 20000,
  "data": [
    {
      "id": 1,
      "currentVehicleType": "C1",
      "requestedVehicleType": "C1,C2",
      "applyReason": "已取得C2教练资格证",
      "status": 0,
      "auditReason": null,
      "applyTime": "2026-06-03T10:30:00",
      "auditTime": null
    }
  ]
}
```

#### 返回字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| id | int | 申请ID |
| currentVehicleType | String | 当前准教车型（提交时的快照） |
| requestedVehicleType | String | 申请的新准教车型 |
| applyReason | String | 申请理由 |
| status | int | 0-待审核, 1-已通过, 2-已拒绝 |
| auditReason | String | 拒绝原因（通过时为 null） |
| applyTime | datetime | 申请提交时间 |
| auditTime | datetime | 审核时间（待审核时为 null） |

#### 前端展示建议

根据 `status` 显示不同状态的标签：

| status | 显示 |
|--------|------|
| 0 | 待审核（灰色/黄色标签） |
| 1 | 已通过（绿色标签） |
| 2 | 已拒绝（红色标签，可显示 `auditReason`） |

```json
  ]

```

#### 返回字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| id | int | 申请ID |
| coachId | int | 教练ID |

### 2.3 审核申请

管理员审核教练的准教车型变更申请。

通过时：自动更新 `coach.vehicle_type` 为 `requested_vehicle_type` 的值。
拒绝时：需填写拒绝原因，教练的车型不变。

```
PUT /coach-vehicle-applications/{id}/audit?pass=true
```

#### 请求参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | int | 路径参数 | 申请ID |
| pass | boolean | 是 | true=通过, false=拒绝 |
| auditReason | String | 拒绝时必填 | 拒绝原因（通过时不需要传） |

#### 请求示例

**通过申请：**
```
PUT /coach-vehicle-applications/1/audit?pass=true
```

**拒绝申请：**
```
PUT /coach-vehicle-applications/1/audit?pass=false&auditReason=B1教练资格证未通过审核
```

#### 成功响应（20000）

```json
{
  "state": 20000,
  "data": null
}
```

#### 校验规则

| 场景 | 行为 |
|------|------|
| 申请不存在 | 返回 40400 "申请记录不存在" |
| 申请已处理过 | 返回 40900 "该申请已处理，无法重复操作" |
| 拒绝时不填原因 | 返回 40000 "拒绝时必须填写原因" |
| 通过时教练信息已删除 | 返回 40400 "教练信息不存在" |

#### 错误示例

**409 — 重复审核：**
```json
{
  "state": 40900,
  "message": "该申请已处理，无法重复操作",
  "data": null
}
```

## 三、业务流程

### 完整流程

```
教练端                             管理员端
  │                                  │
  ├─ 查看当前车型 (GET /coach-portal/rating)
  │   (返回数据中含 vehicleType)
  │                                  │
  ├─ 提交车型变更申请                  │
  │   POST /coach-portal/            │
  │   vehicle-applications            │
  │   ?requestedVehicleType=C1,C2     │
  │   &applyReason=...               │
  │                                  │
  │                   ┌───────────────┤
  │                   │  查询待审核列表   │
  │                   │  GET /coach-   │
  │                   │  vehicle-       │
  │                   │  applications/  │
  │                   │  pending        │
  │                   ├───────────────┤
  │                   │  审核申请       │
  │                   │  PUT /coach-    │
  │                   │  vehicle-       │
  │                   │  applications/  │
  │                   │  {id}/audit     │
  │                   │  ?pass=true     │
  │                   │                │
  │                   │  通过时自动更新  │
  │                   │  coach.         │
  │                   │  vehicle_type   │
  │                   └──────┬─────────┘
  │                          │
  ├─ 查看申请结果               │
  │  GET /coach-portal/       │
  │  vehicle-applications      │
  │  → 看到 status 变为 1 或 2 │
  │                          │
  │  如已通过：查看当前车型已更新    │
  │  GET /coach-portal/rating  │
  │                          │
```

## 五、前端对接建议

### 教练端页面

**提交申请页：**
- 先调 `GET /coach-portal/rating` 获取当前准教车型展示给教练
- 提供多选或输入框让教练选择/填写想要增加的车型
- 提交前校验：不能跟当前完全一致、不能为空
- 提交后提示结果，跳转到申请记录页

**申请记录页：**
- 调 `GET /coach-portal/vehicle-applications` 展示列表
- 用 `status` 字段区分显示状态标签
- 已拒绝的显示 `auditReason`

### 管理员端页面

**待审核列表页：**
- 调 `GET /coach-vehicle-applications/pending` 展示
- 每条记录显示：教练姓名、当前车型、申请车型、申请理由、申请时间
- 提供"通过"和"拒绝"操作按钮

**审核操作：**
- 通过：直接调 `PUT .../audit?pass=true`，成功后刷新列表
- 拒绝：弹窗要求填写拒绝原因，调 `PUT .../audit?pass=false&auditReason=原因`
- 注意防重复点击：已处理的申请不能再操作

**历史记录页：**
- 调 `GET /coach-vehicle-applications?page=1&size=10` 分页展示全部记录
- 可用 `status` 筛选各状态记录
