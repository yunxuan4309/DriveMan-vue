## 通用说明

### 基础信息

| 项目 | 值 |
|------|-----|
| Base URL | `http://localhost:9500` |
| Token 传递 | `Authorization: Bearer <token>` |
| 响应格式 | `JsonResult<T>` |

### 统一响应结构

```json
{
  "state": 20000,
  "message": null,
  "data": { ... }
}
```

### 前端响应拦截器处理规则

- `state === 20000` → 拦截器自动解包，return `body.data`
- `state !== 20000` → 拦截器调用 `handleBusinessError()`，显示 `body.message`，return `Promise.reject()`
- HTTP 4xx/5xx → 拦截器显示 `ElMessage.error(body.message)`

### 前端发请求时无需判断 state

拦截器已处理，请求成功直接拿到 `data`，错误由拦截器统一弹提示。

### 分页响应结构（MyBatis-Plus Page）

```json
{
  "records": [ ... ],
  "total": 36,
  "size": 10,
  "current": 1,
  "pages": 4
}
```

### 角色权限

| 角色 | role 值 |
|------|---------|
| 学员 | 1 |
| 教练 | 2 |
| 管理员 | 3 |

## 1. 考试场次查询

学员申请合场时，需要先选择一个考试场次（科目二/科目三）。

### 请求

```
GET /exam-sessions
```

### 请求参数（Query）

| 参数名 | 类型 | 必填 | 默认值 | 说明 |
|--------|------|------|--------|------|
| page | int | 否 | 1 | 页码 |
| size | int | 否 | 10 | 每页条数 |
| subject | int | 否 | — | 科目筛选：2-科目二(场地驾驶), 3-科目三(道路驾驶) |
| licenseType | string | 否 | — | 车型筛选：C1/C2/B1/C5 等 |
| examDateStart | string | 否 | — | 考试日期起始，格式 yyyy-MM-dd |
| examDateEnd | string | 否 | — | 考试日期截止，格式 yyyy-MM-dd |
| location | string | 否 | — | 考试地点模糊搜索 |

> **注意：** 合场仅支持科目二和科目三，前端下拉框应只显示"科目二"(value=2) 和 "科目三"(value=3)。

### 响应 data（Page<ExamSession>）

```json
{
  "records": [
    {
      "id": 1,
      "subject": 2,
      "licenseType": "C1",
      "examDate": "2026-06-15",
      "startTime": "08:30:00",
      "location": "南坪科目二考场",
      "venueId": null,
      "totalQuota": 80,
      "remainingQuota": 78,
      "status": 1,
      "version": 1,
      "createTime": "2026-06-10T10:00:00",
      "updateTime": "2026-06-10T10:00:00"
    }
  ],
  "total": 36,
  "size": 10,
  "current": 1,
  "pages": 4
}
```

### ExamSession 字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| id | int | 场次ID（申请合场时传入） |
| subject | int | 科目：1-4 |
| licenseType | string | 适用车型 |
| examDate | string | 考试日期 yyyy-MM-dd |
| startTime | string | 开始时间 HH:mm:ss |
| location | string | 考试地点 |
| totalQuota | int | 总名额 |
| remainingQuota | int | 剩余名额 |
| status | int | 1-报名中, 2-已满, 3-已结束 |

## 3. 申请合场

学员选择一个考试场次和用车类型，系统自动定价并生成待支付账单。

### 请求

```
POST /familiarizations/apply
```

### 请求参数（Query）

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| examSessionId | int | 是 | 考试场次ID（从场次列表中选择） |
| carType | int | 是 | 用车类型：1-教练车（需绑定教练陪同）, 2-考试车（考场提供陪练） |

### 业务校验规则

| 规则 | 失败提示 |
|------|---------|
| 学员不存在或角色不是学员 | "学员不存在" |
| 学员未通过审核 | "学员尚未通过审核" |
| 场次不存在 | "考试场次不存在" |
| 场次名额已满 | "该场次名额已满" |
| 车型不匹配 | "该场次仅限 X 车型，您的车型为 Y" |
| 科目一或科目四 | "科目X为理论考试，无需合场练习。只有科目二（场地驾驶）和科目三（道路驾驶）需要合场" |
| 教练车模式但未绑定教练 | "您当前没有绑定的教练，无法使用教练车合场" |
| carType 不是 1 或 2 | "用车类型非法: 1-教练车, 2-考试车" |
| 未配置合场费用 | "未配置 X 科目X 的合场费用（合场(教练车/考试车)），请管理员在【费用标准管理 → fee-standards】页面..." |

### 响应 data（Map — 合场详情，含姓名）

```json
{
  "id": 1,
  "studentId": 4,
  "examSessionId": 2,
  "subject": 2,
  "carType": 1,
  "coachId": 1,
  "amount": 200.00,
  "paymentRecordId": 5,
  "status": 0,
  "scheduledTime": null,
  "payTime": null,
  "remark": null,
  "createTime": "2026-06-11T14:30:00",
  "updateTime": "2026-06-11T14:30:00",
  "isDeleted": 0,
  "exam_date": "2026-06-15",
  "start_time": "08:30:00",
  "venue_name": "南坪科目二考场",
  "license_type": "C1",
  "student_name": "王小明",
  "coach_name": "张教练"
}
```

### 返回字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| id | int | 合场记录ID |
| studentId | int | 学员 user_id |
| examSessionId | int | 考试场次ID |
| subject | int | 科目 |
| carType | int | 1-教练车, 2-考试车 |
| coachId | int | 教练 coach_id（教练车模式） |
| amount | BigDecimal | 费用金额 |
| paymentRecordId | int | 关联支付记录ID（用于支付） |
| status | int | 0-待支付 |
| scheduledTime | string/null | 预约时间 |
| payTime | string/null | 支付时间 |
| student_name | string | 学员姓名 |
| coach_name | string | 教练姓名（教练车模式有值，考试车模式为null） |
| exam_date | string | 考试日期 |
| venue_name | string | 考试地点 |

> **注意：** 返回的 Map 中字段为下划线命名（exam_date, start_time），与驼峰的实体字段混用。这是 MyBatis 返回 Map 的默认行为，前端直接按下划线字段名取值即可。


| 规则 | 失败提示 |

| 合场记录不存在 | "合场记录不存在" |

| 状态不是待支付 | "当前状态不允许支付" |

## 5. 我的合场记录（分页）

分页查询自己的合场记录列表，支持按状态和时间范围筛选。

### 请求

```
GET /familiarizations/my
```

### 请求参数（Query）

| 参数名 | 类型 | 必填 | 默认值 | 说明 |
|--------|------|------|--------|------|
| page | int | 否 | 1 | 页码 |
| size | int | 否 | 10 | 每页条数 |
| status | int | 否 | — | 状态筛选：0-待支付, 1-已支付, 2-已完成, 3-已取消 |
| startDate | string | 否 | — | 创建时间起始（含），格式 yyyy-MM-dd HH:mm:ss |
| endDate | string | 否 | — | 创建时间结束（含），格式 yyyy-MM-dd HH:mm:ss |

### 响应 data（Page<Map>）

```json
{
  "records": [
    {
      "id": 2,
      "studentId": 4,
      "examSessionId": 2,
      "subject": 2,
      "carType": 1,
      "coachId": 1,
      "amount": 200.00,
      "paymentRecordId": 6,
      "status": 1,
      "scheduledTime": null,
      "payTime": "2026-06-11T14:35:00",
      "remark": null,
      "createTime": "2026-06-10T10:00:00",
      "updateTime": "2026-06-11T14:35:00",
      "isDeleted": 0,
      "exam_date": "2026-06-15",
      "start_time": "08:30:00",
      "venue_name": "南坪科目二考场",
      "license_type": "C1",
      "student_name": "王小明",
      "coach_name": "张教练"
    }
  ],
  "total": 2,
  "size": 10,
  "current": 1,
  "pages": 1
}
```

> **注意：** 该接口原来是返回 List（无分页），**现已改为 Page 格式**。如果前端之前调用的是旧版，需要修改为分页组件调用方式。

### 前端 Vue 调用示例

```js
// api/familiarization.js
export function getMyFamiliarizations(params) {
  return request({
    url: '/familiarizations/my',
    method: 'get',
    params
  })
}

// 页面组件中
const listParams = reactive({
  page: 1,
  size: 10,
  status: undefined,  // 可选
  startDate: undefined,
  endDate: undefined
})

const fetchList = async () => {
  const res = await getMyFamiliarizations(listParams)
  // 拦截器已解包，res 就是 Page 对象
  tableData.value = res.records
  total.value = res.total
}
```

## 8. 管理员-安排合场时间

为已支付的合场记录安排具体时间。

### 请求

```
PUT /familiarizations/{id}/schedule?scheduledTime=2026-06-16 09:30:00
```

### 路径参数

| 参数 | 说明 |
|------|------|
| id | 合场记录ID |

### 请求参数（Query）

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| scheduledTime | string | 是 | 预约时间，格式 yyyy-MM-dd HH:mm:ss（注意是空格不是T） |

### 业务校验规则

| 规则 | 失败提示 |
|------|---------|
| 记录不存在 | "合场记录不存在" |
| 状态不是已支付(1) | "仅已支付的合场可以安排时间" |

### 响应 data（Map）

返回更新后的合场详情（与申请接口结构一致），此时 `scheduledTime` 有值。

| 字段 | 类型 | 说明 |

> 金额字段 `amount` 保留两位小数，可直接显示为 "200.00"。

## 附录：合场状态流转

```
0(待支付) ──支付──→ 1(已支付) ──安排时间──→ 1(已支付) ──完成──→ 2(已完成)
    │                    │                                             │
    └──取消──→ 3(已取消)  └──取消──→ 3(已取消)               不可取消
```

### 各状态说明

| 状态 | 值 | 含义 | 可执行操作 |
|------|----|------|-----------|
| 待支付 | 0 | 刚申请，未付款 | 支付、取消 |
| 已支付 | 1 | 已付款，管理员可安排时间 | 安排时间、标记完成、取消 |
| 已完成 | 2 | 合场练习完成 | 无 |
| 已取消 | 3 | 已取消 | 无 |

### 前端状态标签建议

```js
const statusMap = {
  0: { text: '待支付', type: 'warning' },
  1: { text: '已支付', type: 'success' },
  2: { text: '已完成', type: 'primary' },
  3: { text: '已取消', type: 'info' }
}

const carTypeMap = {
  1: { text: '教练车', icon: 'Coach' },
  2: { text: '考试车', icon: 'Exam' }
}
```
