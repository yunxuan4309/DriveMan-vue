# 增驾申请管理 — 前端接口文档

## 业务流程全景

```
学员已持有驾照（如 C2），想增驾到其他车型（如 C1 或 B1）
      │
      ▼
① 学员提交增驾申请
   POST /license-upgrades/apply
      │
      ├── 同级增驾（如 C2 → C1）：无需驾驶证材料
      │
      └── 升级增驾（如 C1 → B1）：需先上传驾驶证材料，再提交申请
            │
            ▼
      ② 管理员审核
         PUT /license-upgrades/{id}/audit
            │
            ├── 通过 → 管理员安排考试
            │
            └── 不通过 → 流程终止
                  │
                  ▼
            ③ 管理员录入考试成绩
               PUT /license-upgrades/{id}/exam-result
                  │
                  ├── 通过 → 自动更新学员的准驾车型
                  │
                  └── 不通过 → 流程终止（学员可重新申请）
```

---

## 通用说明

### 统一响应格式

```json
{
  "state": 20000,
  "message": null,
  "data": { ... }
}
```

| state | 含义 |
|-------|------|
| 20000 | 成功 |
| 40000 | 请求参数错误 |
| 40100 | 未登录或 token 过期 |
| 40300 | 权限不足 |
| 40400 | 资源不存在 |
| 40900 | 数据冲突 |

### 认证方式

所有接口需要在请求头中携带 Token：

```
Authorization: Bearer <token>
```

### 角色说明

| 值 | 角色 | 操作权限 |
|----|------|---------|
| 1 | 学员 | 提交增驾申请、查看自己的申请记录 |
| 3 | 管理员 | 查看全部申请、审核、录入考试成绩 |

## 接口列表

### 1. 学员提交增驾申请

学员提交增驾申请，需指定目标准驾车型和增驾类型。

```
POST /license-upgrades/apply
```

**请求参数：**

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| targetLicense | String | 是 | 目标准驾车型，如 C1, C6, B1, A1 等 |
| upgradeType | Integer | 是 | 1-同级增驾, 2-升级增驾 |
| licenseFileId | Integer | 否 | 驾驶证材料文件 ID（升级增驾时必填） |

**响应：**

```json
{
  "state": 20000,
  "message": null,
  "data": {
    "id": 1,
    "studentId": 4,
    "originalLicense": "C2",
    "targetLicense": "C1",
    "upgradeType": 1,
    "status": 0,
    "remark": null,
    "examStatus": 0,
    "examRemark": null,
    "licenseFileId": null,
    "createTime": "2026-06-04T10:00:00"
  }
}
```

**说明：**
- 系统会自动从当前登录学员的 `license_type` 读取原准驾车型
- 同一学员不能同时有多个进行中的申请（待审核或审核通过中）
- 增驾类型区分：
  - **同级增驾**（如 C2→C1, B1→B2）：同一级别内切换，无需驾驶证材料
  - **升级增驾**（如 C1→B1, C1→A1）：跨级别升级，**必须上传驾驶证材料**
- 升级增驾时，需先调用文件上传接口获取 `licenseFileId`，再提交申请

#### 车型等级分组

| 分组 | 包含车型 |
|------|---------|
| 小型汽车 | C1, C2, C5, C6 |
| 中型/大型货车 | B1, B2 |
| 大型客车/城市公交 | A1, A3 |
| 摩托车 | D, E, F |
| 特种车辆 | M, N, P |

同级增驾 = 在同一分组内切换。

### 2. 学员查看自己的增驾申请

```
GET /license-upgrades/my
```

**请求参数：** 无（通过 Token 识别当前用户）

**响应：**

```json
{
  "state": 20000,
  "data": [
    {
      "id": 1,
      "studentId": 4,
      "originalLicense": "C2",
      "targetLicense": "C1",
      "upgradeType": 1,
      "status": 1,
      "remark": "审核通过，安排考试",
      "examStatus": 0,
      "examRemark": null,
      "licenseFileId": null,
      "createTime": "2026-06-04T10:00:00",
      "updateTime": "2026-06-05T14:30:00"
    }
  ]
}
```

**返回字段说明：**

| 字段 | 类型 | 说明 |
|------|------|------|
| id | Integer | 申请记录 ID |
| studentId | Integer | 学员 ID |
| originalLicense | String | 原准驾车型 |
| targetLicense | String | 目标准驾车型 |
| upgradeType | Integer | 1-同级增驾, 2-升级增驾 |
| status | Integer | 0-待审核, 1-审核通过, 2-审核不通过 |
| remark | String | 审核备注 |
| examStatus | Integer | 0-待考试, 1-考试通过, 2-考试不通过 |
| examRemark | String | 考试不通过原因 |
| licenseFileId | Integer | 驾驶证材料文件 ID |
| createTime | String | 创建时间 |
| updateTime | String | 修改时间 |

### 4. 管理员审核增驾申请

管理员审核学员的增驾申请。

```
PUT /license-upgrades/{id}/audit
```

**路径参数：**

| 参数名 | 类型 | 说明 |
|--------|------|------|
| id | Integer | 增驾申请记录 ID |

**请求参数：**

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| status | Integer | 是 | 1-审核通过, 2-审核不通过 |
| remark | String | 否 | 审核备注 |

**响应：**

```json
{
  "state": 20000,
  "message": null,
  "data": null
}
```

**说明：**
- 仅管理员可调用
- 只能审核状态为「待审核」的申请
- 审核**通过** → 状态变为「审核通过待考试」，**不会立即更新学员车型**
- 审核**不通过**时建议填写备注原因
- 管理员审核时应核验：
  - 学员上传的驾驶证材料是否真实有效
  - 持有年限是否满足要求（升级增驾时）

---

### 5. 管理员录入增驾考试成绩

学员完成增驾考试后，管理员录入考试结果。考试通过后系统自动更新学员的准驾车型。

```
PUT /license-upgrades/{id}/exam-result
```

**路径参数：**

| 参数名 | 类型 | 说明 |
|--------|------|------|
| id | Integer | 增驾申请记录 ID |

**请求参数：**

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| examStatus | Integer | 是 | 1-考试通过, 2-考试不通过 |
| examRemark | String | 否 | 考试不通过原因/备注 |

**响应：**

```json
{
  "state": 20000,
  "message": null,
  "data": null
}
```

**说明：**
- 仅管理员可调用
- 必须先审核通过（status=1）才能录入考试成绩
- 考试**通过**后：
  - 系统自动将学员的 `license_type` 更新为目标车型
  - 学员后续报名考试、约课等均使用新车型
- 考试**不通过**后流程终止，学员可重新提交申请

## 后端校验规则说明（前端可参考做提示）

### 1. 升级路径校验

并非所有车型之间都能增驾，合法的升级路径如下（依据公安部第172号令）：

| 目标车型 | 来源车型 | 最低持有年限 |
|---------|---------|------------|
| C6 | C1, C2 | 1年 |
| B1 | A3, B2, C1, C2, C3, C4 | 2年 |
| A2 | B1, B2 | 2年 |
| A2 | A1 | 1年 |
| A1 | A3, B1 | 2年 |
| A1 | B2 | 3年 |
| A1 | A2 | 1年 |
| A3 | C1, C2 | 1年 |
| B2 | C1, C2 | 1年 |
| D | E, F | 1年 |
| E | F | 1年 |

### 2. 年龄校验

| 车型 | 最低年龄 |
|------|---------|
| C1, C2, C5 | 18岁 |
| C6, B1, B2, A3 | 20岁 |
| A1, A2 | 22岁 |
| D, E, F, M | 18岁 |
| N, P | 20岁 |

### 3. 持有年限由人工核验

系统不自动判断持有年限，由管理员在审核时通过查看学员上传的驾驶证材料来判断。

## 与文件系统的关联

升级增驾需要上传驾驶证照片/扫描件，流程如下：

1. 先调用文件上传接口获取 `fileId`：

```
POST /files/upload
Content-Type: multipart/form-data

file: (驾驶证照片/扫描件)
bizType: license_upgrade
```

2. 再将 `fileId` 传入 `POST /license-upgrades/apply` 的 `licenseFileId` 参数
