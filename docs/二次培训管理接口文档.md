# 二次培训（补考培训）管理 — 前端接口文档

## 业务流程全景

```
学员挂科（exam_registration.passStatus=0）
      │
      ▼
① 学员申请二次培训
   POST /retake-trainings
      │
      ├── 全包学员（报名时已支付全包套餐）
      │     └── 自动通过，免缴费，直接进入培训中状态
      │
      └── 非全包学员
            │
            ▼
      ② 管理员审核
         PUT /retake-trainings/{id}/audit
            │
            ├── 通过 → 生成 training_fee 账单（待缴费）
            │         → 学员在「我的账单」中缴费
            │         → 缴费后状态变为「培训中」
            │
            └── 不通过 → 关闭
      │
      ▼
③ 教练安排培训 → 标记完成
   PUT /retake-trainings/{id}/complete
      │
      ▼
④ 学员重新报名考试（走考试报名流程）
   POST /exam-registrations
```

### 全包与非全包的区别

| | 全包学员 | 非全包学员 |
|:---|:---|:---|
| 二次培训费 | 免费（套餐已含） | 需缴费（默认 300 元，管理员可调整） |
| 申请流程 | 自动通过，无需审核 | 需管理员审核 + 设定费用 |
| 账单生成 | 不生成 | 审核通过后自动生成 `training_fee` 账单 |
| 缴费 | 无需 | 需在「我的账单」中完成支付 |

### 系统如何判断全包

学员报名审核通过时，系统会根据其车型自动生成一笔 `registration_fee` 账单（全包套餐价）。**如果该账单已支付 → 全包学员。** 判断逻辑在服务端自动完成，前端无需传参。

```json
{

  "message": null,

```

| state | 含义 |
|-------|------|

所有接口需要在请求头中携带 Token：


### 角色说明

| 值 | 角色 | 操作权限 |
|----|------|---------|
| 1 | 学员 | 申请二次培训、查看自己的记录 |
| 2 | 教练 | 查看名下学员的二次培训记录（只读）、标记完成 |
| 3 | 管理员 | 审核、查看全部记录、设定培训费 |

## 接口列表

### 1. 学员申请二次培训

学员对已挂科的考试申请二次培训。全包学员自动通过免缴费；非全包学员需管理员审核。

```
POST /retake-trainings
```

**请求参数：**

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| examRegistrationId | Integer | 是 | 挂科的考试报名记录 ID（passStatus=0） |
| coachId | Integer | 否 | 希望指派的教练 ID（可选，不填则由系统分配） |
| reason | String | 否 | 申请说明 |

**响应：**

```json
{
  "state": 20000,
  "message": null,
  "data": null
}
```

**说明：**
- 只能对自己已挂科的考试申请（`passStatus = 0`），未出成绩或已合格的无法申请
- 同一考试报名记录不可重复提交二次培训申请
- 全包学员 → 自动通过，`status=1`（培训中），无需审核和缴费
- 非全包学员 → `status=0`（待审核），等待管理员处理

---

### 2. 管理员审核二次培训申请

管理员审核非全包学员的二次培训申请并设定培训费。

```
PUT /retake-trainings/{id}/audit
```

**路径参数：**

| 参数名 | 类型 | 说明 |
|--------|------|------|
| id | Integer | 二次培训记录 ID |

**请求参数：**

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| pass | boolean | 是 | true-通过, false-不通过 |
| amount | BigDecimal | 否 | 培训费金额（元），为空则从系统配置读取默认值（300 元） |

**响应：**

```json
{
  "state": 20000,
  "message": null,
  "data": null
}
```

**说明：**
- 仅管理员可调用
- 全包学员无需审核（申请时已自动通过），对此类记录审核会返回错误
- `amount` 不传时使用 `config` 表中的 `retake_training_fee`（默认 300 元）
- 审核通过后自动生成 `bizType=training_fee` 的待支付账单，学员需在「支付管理」中完成缴费

---

### 3. 完成培训（教练标记）

教练标记二次培训为已完成。非全包学员需先确认已缴费。

```
PUT /retake-trainings/{id}/complete
```

**路径参数：**

| 参数名 | 类型 | 说明 |
|--------|------|------|
| id | Integer | 二次培训记录 ID |

**响应：**

```json
{
  "state": 20000,
  "message": null,
  "data": null
}
```

**说明：**
- 仅教练可调用
- 非全包学员如果未缴费，会返回错误提示
- 标记完成后学员可以重新报名考试

---

### 4. 取消二次培训申请

取消二次培训记录（学员或管理员均可）。

```
PUT /retake-trainings/{id}/cancel
```

**路径参数：**

| 参数名 | 类型 | 说明 |
|--------|------|------|
| id | Integer | 二次培训记录 ID |

**响应：**

```json
{
  "state": 20000,
  "message": null,
  "data": null
}
```

**说明：**
- 已完成的培训无法取消

---

### 6. 教练查看名下学员二次培训记录（只读）

```
GET /coach-portal/retake-trainings
```

- 仅教练可调用

## 培训费配置说明

### 默认培训费

培训费默认值从系统配置表读取，管理员可通过以下 SQL 设置：

```sql
-- 查看当前默认值
SELECT * FROM config WHERE config_key = 'retake_training_fee';

-- 修改默认值
UPDATE config SET config_value = '500' WHERE config_key = 'retake_training_fee';
```

默认值为 **300 元**。

### 审核时设定金额

管理员在审核非全包学员的申请时，可通过 `amount` 参数手动设定培训费。不传则使用默认值。

---

## 支付关联

二次培训审核通过后，会自动生成一笔 `bizType=training_fee` 的支付记录。学员通过现有支付流程完成缴费：

```
PUT /payment-records/{paymentId}/pay   学员端支付
PUT /payment-records/{paymentId}/my-pay   或学员端模拟支付
```

收入看板中「当月收入来源」饼图会增加「二次培训费」分类。
