# 支付管理 — 前端接口文档

## 业务流程全景

```
自动生成账单                          学员端                          管理员端
  │                                     │                              │
  │  ① 报名审核通过                     │                              │
  │    → 自动生成报名费账单              │                              │
  │    → payment_record(待支付)         │                              │
  │                                     │                              │
  │  ② 考试报名审核通过                 │                              │
  │    → 自动生成考试费账单              │                              │
  │    → payment_record(待支付)         │                              │
  │                                     │                              │
  │  ③ 学员申请合场                     │                              │
  │    → 自动生成合场费账单              │                              │
  │    → payment_record(待支付)         │                              │
  │                                     │                              │
  │                       ├─ ④ 查看我的账单 ──→                      │
  │                       │   GET /payment-records/my                │
  │                       │   (列出所有待支付/已支付)                 │
  │                       │                                           │
  │                       ├─ ⑤ 支付账单  ────→                      │
  │                       │   PUT /payment-records/{id}/my-pay       │
  │                       │   (模拟支付，一点即付)                    │
  │                       │                                           │
  │                       │                ←── ⑥ 查看欠费清单 ──→   │
  │                       │                     GET /payment-records/ │
  │                       │                         outstanding      │
  │                       │                                           │
  │                       │                ←── ⑦ 确认支付 ──────→   │
  │                       │                     PUT /payment-records/ │
  │                       │                         {id}/pay         │
  │                       │                     (管理员代操作)        │
  │                       │                                           │
  │                       │                ←── ⑧ 退款 ──────────→   │
  │                       │                     PUT /payment-records/ │
  │                       │                         {id}/refund      │
```

---

## 通用说明

### 统一响应格式

```json
{
  "state": 20000,
  "message": null,
  "data": { ... }
}
```

| state | 含义 |
|-------|------|
| 20000 | 成功 |
| 40000 | 请求参数错误 |
| 40300 | 权限不足 |
| 40400 | 资源不存在 |
| 40900 | 数据冲突 |

### 认证方式

```
Authorization: Bearer <token>
```

### 角色说明

| 值 | 角色 |
|----|------|
| 1 | 学员 |
| 2 | 教练 |
| 3 | 管理员 |

基础路径：`/payment-records`

---

## 业务类型枚举（biz_type）

| bizType | 含义 | 生成方式 |
|---------|------|---------|
| `registration_fee` | 报名费 | 报名审核通过时自动生成 |
| `exam_fee` | 考试费 | 考试报名审核通过时自动生成 |
| `familiarization_fee` | 合场费 | 申请合场时自动生成 |
| `other` | 其他 | 管理员手动创建 |

## 支付状态枚举

| status | 含义 |
|--------|------|
| 0 | 待支付 |
| 1 | 已支付 |
| 2 | 已退款 |

## 学员端接口

### 1. 查看我的账单

```
GET /payment-records/my
```

> 仅学员可调用。返回当前学员的所有支付记录（待支付/已支付/已退款）。

**响应 data：**
```json
[
  {
    "id": 1,
    "studentId": 4,
    "bizType": "registration_fee",
    "bizId": 1,
    "amount": 3980.00,
    "status": 0,
    "remark": "C1 C1全包套餐（含补考费）",
    "payTime": null,
    "refundTime": null,
    "createTime": "2026-06-04T10:00:00"
  },
  {
    "id": 2,
    "studentId": 4,
    "bizType": "exam_fee",
    "bizId": 1,
    "amount": 200.00,
    "status": 1,
    "remark": "C1 科目2 科目二模拟训练费",
    "payTime": "2026-06-04T10:30:00",
    "refundTime": null,
    "createTime": "2026-06-04T09:00:00"
  }
]
```

### 2. 支付我的账单

```
PUT /payment-records/{id}/my-pay
```

> 仅学员可调用，且只能支付自己的账单（模拟支付，一点即付）。

**校验逻辑：**
- 账单必须属于当前学员
- 账单状态必须为 0（待支付）

**响应 data：** PaymentRecord 对象（status → 1, payTime 更新）

## 管理员端接口

### 3. 创建支付记录

```
POST /payment-records
```

> 仅管理员可调用。手动创建一笔待支付记录（补录场景）。

**请求体（JSON）：**
```json
{
  "studentId": 4,
  "bizType": "other",
  "amount": 500.00,
  "remark": "补交材料费"
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| studentId | Integer | 是 | 学员ID |
| bizType | String | 否 | 默认 other |
| bizId | Integer | 否 | 关联业务ID |
| amount | BigDecimal | 是 | 金额 |
| remark | String | 否 | 备注 |

### 4. 支付记录列表

```
GET /payment-records
```

> 仅管理员可调用。可按条件筛选查询。

**请求参数（均为可选）：**

| 参数 | 类型 | 说明 |
|------|------|------|
| studentId | Integer | 按学员筛选 |
| bizType | String | 按业务类型筛选 |
| status | Integer | 按状态筛选 |

**响应 data：** `List<PaymentRecord>`

### 5. 确认支付（管理员代操作）

```
PUT /payment-records/{id}/pay
```

> 仅管理员可调用。将待支付记录标记为已支付。

### 6. 退款

```
PUT /payment-records/{id}/refund
```

> 仅管理员可调用。只能对已支付的记录进行退款。

### 7. 欠费清单

```
GET /payment-records/outstanding
```

> 仅管理员可调用。列出所有待支付记录（含学员信息）。

**响应 data：**
```json
[
  {
    "id": 1,
    "studentId": 4,
    "bizType": "registration_fee",
    "bizId": 1,
    "amount": 3980.00,
    "status": 0,
    "remark": "C1 C1全包套餐（含补考费）",
    "createTime": "2026-06-04T10:00:00",
    "student_name": "王小明",
    "student_phone": "15912340001",
    "license_type": "C1"
  }
]
```

**说明：** 返回的 Map 中除了 payment_record 的字段外，额外包含：
- `student_name` — 学员姓名
- `student_phone` — 手机号
- `license_type` — 报考车型

## 账单自动生成触发点

| 业务动作 | 自动生成 | 关联 bizType |
|---------|---------|-------------|
| 管理员审核通过学员报名（PUT /registrations/{userId}/audit?pass=true） | 报名费账单 | registration_fee |
| 管理员审核通过考试报名（PUT /exam-registrations/{id}/audit?pass=true） | 考试费账单 | exam_fee |
| 学员申请合场（POST /familiarizations/apply） | 合场费账单 | familiarization_fee |

> 自动生成的账单均为待支付状态，金额从 fee_standard 表按车型+科目匹配。
