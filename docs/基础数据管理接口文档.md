# 基础数据管理

涵盖以下四个基础数据模块：

| 模块 | 基础路径 | 角色 |
|------|---------|------|
| 费用标准管理 | `/fee-standards` | 管理员 |
| 车型科目配置 | `/license-configs` | 管理员 |
| 考场信息管理 | `/exam-venues` | 管理员 |
| 特种车辆考试管理 | `/special-exams` | 管理员/学员/教练 |

## 通用说明

### 统一响应格式

```json
{
  "state": 20000,       // 业务状态码
  "message": null,      // 错误时携带提示信息
  "data": { ... }       // 成功时携带数据
}
```

| state | 含义 |
|-------|------|
| 20000 | 成功 |
| 40000 | 请求参数错误 |
| 40100 | 未登录或 token 过期 |
| 40300 | 权限不足 |
| 40400 | 资源不存在 |
| 40900 | 数据冲突（如用户名已存在） |

### 认证方式

所有接口（除登录和注册外）需要在请求头中携带 Token：

```
Authorization: Bearer <token>
```

### 角色说明

| 值 | 角色 |
|----|------|
| 1 | 学员 |
| 2 | 教练 |
| 3 | 管理员 |

## 一、费用标准管理

基础路径：`/fee-standards`

> 仅管理员可调用。

### 费用标准数据结构

```json
{
  "id": 1,
  "licenseType": "C1",        // 车型
  "subject": 2,               // 科目 1-4，null 表示全包套餐
  "amount": 200.00,           // 金额(元)
  "description": "科目二模拟训练费"
}
```

### 1.1 查询所有费用标准

```
GET /fee-standards
```

返回全部费用标准列表，按车型和科目升序排列。

### 1.2 按车型查询

```
GET /fee-standards/type/{licenseType}
```

例：`GET /fee-standards/type/C1` 查询 C1 车型的所有费用记录。

### 1.3 查询费用详情

```
GET /fee-standards/{id}
```

### 1.4 新增费用标准

```
POST /fee-standards
Content-Type: application/json

{
  "licenseType": "C1",
  "subject": 2,
  "amount": 800.00,
  "description": "科目二 场地驾驶技能培训费"
}
```

> 全包套餐 subject 传 null。

### 1.5 修改费用标准

```
PUT /fee-standards/{id}
Content-Type: application/json

{
  "amount": 900.00
}
```

### 1.6 删除费用标准

```
DELETE /fee-standards/{id}
```

## 二、车型科目配置管理

基础路径：`/license-configs`

> 仅管理员可调用。

### 车型配置数据结构

```json
{
  "id": 1,
  "licenseType": "C1",                // 车型
  "subject": 2,                       // 科目 1-4（小汽车）或 1-2（特种车辆）
  "requiredHours": 16,                // 要求学时
  "examItems": "倒车入库,侧方停车,...", // 考试项目，逗号分隔
  "description": "科目二 场地驾驶技能",
  "sortOrder": 2,                     // 排序
  "examMode": 1,                      // 考试模式: 1-小汽车, 2-特种车辆
  "coachAuditRequired": 1,            // 是否需要教练审核: 1-需要, 0-不需要（当前统一走管理员审核）
  "certName": null                    // 获证名称（仅特种车辆，如"叉车操作证"）
}
```

### 考试模式字段说明

| examMode | 含义 | 科目范围 |
|----------|------|---------|
| 1 | 小汽车模式 | subject 1-4（科一~科四） |
| 2 | 特种车辆模式 | subject 1=理论, 2=实操 |

### 2.1 查询所有车型配置

```
GET /license-configs
```

返回全部车型的科目配置。

### 2.2 按车型查询配置

```
GET /license-configs/type/{licenseType}
```

例：`GET /license-configs/type/C1` 查询 C1 完整科目配置；`GET /license-configs/type/N1` 查询叉车配置。

### 2.3 查询配置详情

```
GET /license-configs/{id}
```

### 2.4 新增车型配置

```
POST /license-configs
Content-Type: application/json

{
  "licenseType": "N1",
  "subject": 2,
  "description": "叉车实操考试",
  "requiredHours": 20,
  "examItems": "起步,货叉装卸,倒车入库,停放",
  "sortOrder": 2,
  "examMode": 2,
  "coachAuditRequired": 0,
  "certName": "叉车操作证"
}
```

### 2.5 修改车型配置

```
PUT /license-configs/{id}
Content-Type: application/json

{
  "requiredHours": 18
}
```

### 2.6 删除车型配置

```
DELETE /license-configs/{id}
```

## 三、考场信息管理

基础路径：`/exam-venues`

> 仅管理员可调用。

### 考场数据结构

```json
{
  "id": 1,
  "name": "南岸区车管所",        // 考场名称
  "address": "南岸区",           // 详细地址
  "contactPhone": "023-12345678", // 联系电话（可选）
  "capacity": 100,               // 容纳人数（可选）
  "facilities": "配备空调、候考室", // 设施说明（可选）
  "status": 1                    // 状态: 1-启用, 0-停用
}
```

### 3.1 查询所有考场

```
GET /exam-venues
```

返回全部考场列表，按名称排序。

### 3.2 查询考场详情

```
GET /exam-venues/{id}
```

### 3.3 新增考场

```
POST /exam-venues
Content-Type: application/json

{
  "name": "南岸区车管所",
  "address": "南岸区",
  "contactPhone": "023-12345678",
  "capacity": 100,
  "facilities": "配备空调、候考室",
  "status": 1
}
```

### 3.4 修改考场信息

```
PUT /exam-venues/{id}
Content-Type: application/json

{
  "capacity": 120,
  "facilities": "配备空调、候考室、饮水机"
}
```

### 3.5 删除考场

```
DELETE /exam-venues/{id}
```

> 逻辑删除，不影响已关联的考试场次历史数据。

## 四、特种车辆考试管理

基础路径：`/special-exams`

> 管理员管理成绩，学员/教练查询考试记录。特种车辆（N1/N2/N3）的考试记录独立管理，不走普通小汽车的考试报名体系。

### 考试记录数据结构

```json
{
  "id": 1,
  "studentId": 6,               // 学员 user_id
  "licenseType": "N1",          // 车型
  "subject": 2,                 // 科目: 1-理论, 2-实操
  "score": 85,                  // 成绩 0-100
  "passStatus": 1,              // 是否合格: 0-不合格, 1-合格
  "retakeCount": 1,             // 补考次数
  "examDate": "2026-06-10T09:00:00", // 考试时间
  "certNo": null                // 证书编号（双科通过后生成）
}
```

### 4.1 查询某学员的特种车辆考试记录

```
GET /special-exams/student/{studentId}?licenseType=N1
```

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| studentId | path | 是 | 学员 ID |
| licenseType | query | 否 | 车型筛选（N1/N2/N3），不传查全部 |

> 学员、教练、管理员均可调用。

### 4.2 查询所有考试记录

```
GET /special-exams?licenseType=N1
```

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| licenseType | query | 否 | 车型筛选 |

> 仅管理员可调用。

### 4.3 录入考试成绩

```
POST /special-exams
Content-Type: application/json

{
  "studentId": 6,
  "licenseType": "N1",
  "subject": 1,
  "score": 92,
  "passStatus": 1,
  "examDate": "2026-06-10T09:00:00"
}
```

> 仅管理员可调用。同一位学员同一车型同一科目只能有一条记录，重复录入会返回 409 错误。

### 4.4 更新考试成绩

```
PUT /special-exams/{id}
Content-Type: application/json

{
  "score": 95,
  "passStatus": 1
}
```

> 仅管理员可调用。每次更新成绩时，补考次数自动 +1。

**学员可调用：**
| 接口 | 用途 |
|------|------|
| `GET /special-exams/student/{studentId}` | 查看自己的特种车辆考试成绩 |

**教练可调用：**
| 接口 | 用途 |
|------|------|
| `GET /special-exams/student/{studentId}` | 查看名下学员的特种车辆考试成绩 |

**管理员可调用：**
| 接口 | 用途 |
|------|------|
| `GET/POST/PUT/DELETE /fee-standards/**` | 费用标准管理 |
| `GET/POST/PUT/DELETE /license-configs/**` | 车型科目配置管理 |
| `GET/POST/PUT/DELETE /exam-venues/**` | 考场信息管理 |
| `GET/POST/PUT /special-exams/**` | 特种车辆考试管理 |
