# 合场管理 — 前端接口文档

## 业务流程全景

```
学员端                             管理员端
  │                                  │
  ├─ ① 查看考试场次 ──────────→     │
  │    GET /exam-sessions            │
  │    (选一个场次申请合场)          │
  │                                  │
  ├─ ② 申请合场 ──────────────→     │
  │    POST /familiarizations/apply  │
  │    (选场次+用车类型)             │
  │    系统自动按 fee_standard 定价  │
  │    生成合场记录+支付账单         │
  │    状态: 待支付                  │
  │                                  │
  ├─ ③ 支付合场 ──────────────→     │
  │    PUT /familiarizations/{id}/   │
  │        pay                       │
  │    状态: 已支付(待安排)          │
  │                                  │
  │                 ←──────── ── ④ 安排时间 │
  │                    PUT /familiarizations/│
  │                        {id}/schedule    │
  │                                  │
  ├─ ⑤ 查看我的合场记录 ─────      │
  │    GET /familiarizations/my      │
  │                                  │
  │                 ←──────── ── ⑥ 标记完成 │
  │                    PUT /familiarizations/│
  │                        {id}/complete    │
  │                                  │
  │                 ←──────── ── ⑦ 取消    │
  │                    PUT /familiarizations/│
  │                        {id}/cancel      │
```

---

## 通用说明

### 统一响应格式

```json
{
  "state": 20000,
  "message": null,
  "data": { ... }
}
```

| state | 含义 |
|-------|------|
| 20000 | 成功 |
| 40000 | 请求参数错误 |
| 40300 | 权限不足 |
| 40400 | 资源不存在 |
| 40900 | 数据冲突 |

### 认证方式

```
Authorization: Bearer <token>
```

### 角色说明

| 值 | 角色 |
|----|------|
| 1 | 学员 |
| 2 | 教练 |
| 3 | 管理员 |

### 用车类型

| carType | 含义 |
|---------|------|
| 1 | 教练车（需绑定教练陪同） |
| 2 | 考试车（考场提供陪练） |

### 合场状态

| status | 含义 |
|--------|------|
| 0 | 待支付 |
| 1 | 已支付（待安排） |
| 2 | 已完成 |
| 3 | 已取消 |

## 学员端接口

### 1. 申请合场

```
POST /familiarizations/apply
```

> 仅学员可调用。系统自动按 fee_standard 定价并生成待支付账单。

**请求参数：**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| examSessionId | Integer | 是 | 考试场次ID |
| carType | Integer | 是 | 用车类型: 1-教练车, 2-考试车 |

**校验逻辑：**
- 学员必须已通过审核（status=1）
- 场次必须存在且有名额
- 车型必须匹配（场次车型 = 学员报考车型）
- 教练车模式：学员必须有当前绑定的教练

**响应 data：**
```json
{
  "id": 1,
  "studentId": 4,
  "examSessionId": 2,
  "subject": 2,
  "carType": 1,
  "coachId": 1,
  "amount": 200.00,
  "paymentRecordId": 5,
  "status": 0,
  "scheduledTime": null,
  "remark": null,
  "createTime": "2026-06-04T10:00:00"
}
```

### 2. 支付合场

```
PUT /familiarizations/{id}/pay
```

> 仅学员可调用，且只能支付自己的合场记录（模拟支付，一点即付）。

**路径参数：**

| 参数 | 类型 | 说明 |
|------|------|------|
| id | Integer | 合场记录ID |

**响应 data：** FamiliarizationRecord 对象（status → 1, payTime 更新）

### 3. 我的合场记录列表

```
GET /familiarizations/my
```

> 仅学员可调用，返回自己的合场记录。

**响应 data：**

## 管理员端接口

### 4. 合场记录列表

```
GET /familiarizations
```

> 仅管理员可调用，查看所有合场记录。

**响应 data：** 同 `GET /my`，但包含所有学员的记录。

### 5. 安排合场时间

```
PUT /familiarizations/{id}/schedule
```

> 仅管理员可调用，为已支付的合场安排具体时间。

**请求参数：**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| scheduledTime | String | 是 | 格式: yyyy-MM-dd HH:mm:ss |

### 6. 标记合场完成

```
PUT /familiarizations/{id}/complete
```

> 仅管理员可调用。仅已支付的合场可以标记完成。

### 7. 取消合场

```
PUT /familiarizations/{id}/cancel
```

> 仅管理员可调用。已完成的合场不能取消。

## fee_standard 合场定价说明

合场费用从 `fee_standard` 表读取，匹配规则：`license_type + subject + description`。

| description 格式 | 含义 |
|-----------------|------|
| `合场(教练车)` | 使用教练车，教练陪同 |
| `合场(考试车)` | 使用考试车，考场提供陪练 |

示例定价（预置种子数据）：

| 车型 | 科目 | 教练车 | 考试车 |
|:---|:---:|---:|---:|
| C1 | 2 | ¥200 | ¥350 |
| C1 | 3 | ¥250 | ¥400 |
| C2 | 2 | ¥200 | ¥350 |
| C2 | 3 | ¥250 | ¥400 |
| N1 | 2 | ¥300 | ¥500 |
| N2 | 2 | ¥350 | ¥550 |
| N3 | 2 | ¥300 | ¥500 |
