# 体检申请管理 — 前端接口文档

## 业务流程全景

```
学员提交体检申请（选择地点和日期）
      │
      ▼
① 管理员审核
   PUT /physical-exams/{id}/audit
      │
      ├── 通过 → 学员去体检
      │
      └── 不通过 → 流程终止
            │
            ▼
      ② 管理员录入体检结果
         PUT /physical-exams/{id}/result
            │
            ├── 合格 → 体检完成
            │
            └── 不合格 → 学员可再次申请
```

### 关于再次申请的说明

- 体检**不合格**的学员，身体养好后可以**再次提交申请**重新体检
- 体检**合格**或**已有进行中**的申请无法再次提交

---

## 通用说明

### 统一响应格式

```json
{
  "state": 20000,
  "message": null,
  "data": { ... }
}
```

| state | 含义 |
|-------|------|
| 20000 | 成功 |
| 40000 | 请求参数错误 |
| 40100 | 未登录或 token 过期 |
| 40300 | 权限不足 |
| 40400 | 资源不存在 |
| 40900 | 数据冲突 |

### 认证方式

所有接口需要在请求头中携带 Token：

```
Authorization: Bearer <token>
```

### 角色说明

| 值 | 角色 | 操作权限 |
|----|------|---------|
| 1 | 学员 | 提交体检申请、查看自己的申请记录 |
| 3 | 管理员 | 查看全部申请、审核、录入体检结果 |

## 接口列表

### 1. 学员提交体检申请

学员选择体检地点和日期，提交体检申请。

```
POST /physical-exams/apply
```

**请求参数：**

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| location | String | 是 | 体检地点 |
| examDate | String | 是 | 预约体检日期，格式 yyyy-MM-dd |

**响应：**

```json
{
  "state": 20000,
  "message": null,
  "data": {
    "id": 1,
    "studentId": 4,
    "location": "南岸区人民医院",
    "examDate": "2026-06-15",
    "status": 0,
    "remark": null,
    "fileId": null,
    "result": null,
    "createTime": "2026-06-04T10:00:00"
  }
}
```

**说明：**
- 同一学员不能同时有多个进行中的申请（待审核或审核通过未完成）
- 上次体检已合格的学员不可再次提交
- 体检日期格式必须为 `yyyy-MM-dd`

---

### 2. 学员查看自己的体检申请

学员查看自己提交的所有体检申请记录及结果。

```
GET /physical-exams/my
```

**请求参数：** 无（通过 Token 识别当前用户）

**响应：**

```json
{
  "state": 20000,
  "data": [
    {
      "id": 1,
      "studentId": 4,
      "location": "南岸区人民医院",
      "examDate": "2026-06-15",
      "status": 3,
      "remark": "请按时前往体检",
      "fileId": 10,
      "result": 1,
      "createTime": "2026-06-04T10:00:00",
      "updateTime": "2026-06-05T14:30:00"
    }
  ]
}
```

**返回字段说明：**

| 字段 | 类型 | 说明 |
|------|------|------|
| id | Integer | 申请记录 ID |
| studentId | Integer | 学员 ID |
| location | String | 体检地点 |
| examDate | String | 预约体检日期 |
| status | Integer | 0-待审核, 1-审核通过, 2-审核不通过, 3-已完成 |
| remark | String | 审核备注（不通过原因等） |
| fileId | Integer | 体检报告文件 ID |
| result | Integer | NULL-未出结果, 0-不合格, 1-合格 |
| createTime | String | 创建时间 |
| updateTime | String | 修改时间 |

### 3. 管理员查询所有体检申请

```
GET /physical-exams
```

**请求参数：** 无

**响应：** 同上（列表）

### 4. 管理员审核体检申请

管理员审核学员的体检申请。

```
PUT /physical-exams/{id}/audit
```

**路径参数：**

| 参数名 | 类型 | 说明 |
|--------|------|------|
| id | Integer | 体检申请记录 ID |

**请求参数：**

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| status | Integer | 是 | 1-审核通过, 2-审核不通过 |
| remark | String | 否 | 审核备注 |

**响应：**

```json
{
  "state": 20000,
  "message": null,
  "data": null
}
```

**说明：**
- 仅管理员可调用
- 只能审核状态为「待审核」的申请
- 审核通过后，学员状态变为「审核通过 — 待体检」
- 审核不通过时建议填写备注原因

---

### 5. 管理员录入体检结果

管理员录入学员的体检结果（体检报告文件和合格/不合格）。

```
PUT /physical-exams/{id}/result
```

**路径参数：**

| 参数名 | 类型 | 说明 |
|--------|------|------|
| id | Integer | 体检申请记录 ID |

**请求参数：**

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| fileId | Integer | 是 | 体检报告文件 ID（需先通过文件上传接口获取） |
| result | Integer | 是 | 0-不合格, 1-合格 |

**响应：**

```json
{
  "state": 20000,
  "message": null,
  "data": null
}
```

**说明：**
- 仅管理员可调用
- 必须先审核通过（status=1）才能录入结果
- 每个申请只能录入一次结果，不可重复覆盖
- 录入后状态自动变为「已完成」

## 状态流转图

```
                         ←――――――――――――――――――┐
                         │                    │
     学员申请            │ 不合格后可再次申请   │
┌──────────┐            │                    │
│ 0:待审核  │           │                    │
└─────┬────┘            │                    │
      │                  │                    │
      ├── 管理员审核通过 ──→ ┌──────────┐     │
      │                  │ 1:审核通过  │     │
      │                  │  (待体检)  │     │
      │                  └─────┬──────┘     │
      │                        │            │
      │                        │ 管理员录入  │
      │                        │ 体检结果    │
      │                        ▼            │
      │                  ┌──────────┐       │
      │                  │ 3:已完成  │       │
      │                  │ result=0 │───────┘
      │                  │ (不合格) │
      │                  └──────────┘
      │
      └── 管理员审核不通过 → ┌──────────┐
                           │ 2:不通过  │
                           └──────────┘

                    ┌──────────┐
                    │ 3:已完成  │
                    │ result=1 │
                    │  (合格)  │ → 不可再次申请
                    └──────────┘
```

## 与文件系统的关联

录入体检结果时需上传体检报告文件，流程如下：

1. 先调用文件上传接口获取 `fileId`：

```
POST /files/upload
Content-Type: multipart/form-data

file: (体检报告 PDF/图片)
bizType: physical_exam
```

2. 再将 `fileId` 传入 `PUT /physical-exams/{id}/result`
