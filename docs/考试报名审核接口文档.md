# 考试报名与审核 — 前端接口文档

## 业务流程全景

```
学员端                         管理员端                         教练端
  │                              │                              │
  ├─ ① 查看考试场次 ──────────→  │                              │
  │    GET /exam-sessions        │                              │
  │                              │                              │
  ├─ ② 报名考试 ──────────────→  │                              │
  │    POST /exam-registrations  │                              │
  │    (自动校验车型/学时/名额)    │                              │
  │                              │                              │
  │                              ├─ ③ 查看报名记录 ──────────→  │
  │                              │    GET /exam-registrations   │
  │                              │    (管理员分页查看所有)       │
  │                              │                              │
  │                              ├─ ④ 审核报名 ──────────────→  │
  │                              │    PUT /exam-registrations/  │
  │                              │        {id}/audit           │
  │                              │    (通过→扣名额+生成准考证)   │
  │                              │    (不通过→状态置为拒绝)      │
  │                              │                              │
  │  ←─── ⑤ 查看审核状态 ────── │                              │
  │    GET /exam-registrations/  │                              │
  │        student/{studentId}   │                              │
  │                              │                              │
  │                              ├─ ⑥ 录入成绩 ──────────────→  │
  │                              │    PUT /exam-registrations/  │
  │                              │        {id}/score           │
  │                              │    (自动判定合格/不合格)      │
  │                              │                              │
  │  ←─── ⑦ 查询成绩 ────────── │           ←─── ⑧ 查看 ─── │
  │    GET /exam-registrations/  │           GET /coach-portal/│
  │        my-scores/{studentId} │           exam-registrations│
  │                              │           (名下学员的报名)   │
  │                              │                              │
```

## 通用说明

### 统一响应格式

```json
{
  "state": 20000,       // 业务状态码
  "message": null,      // 错误时携带提示信息
  "data": { ... }       // 成功时携带数据
}
```

| state | 含义 |
|-------|------|
| 20000 | 成功 |
| 40000 | 请求参数错误 |
| 40100 | 未登录或 token 过期 |
| 40300 | 权限不足 |
| 40400 | 资源不存在 |
| 40900 | 数据冲突（如名额不足、重复操作） |

### 认证方式

除登录注册外，所有接口需要在请求头携带 Token：

```
Authorization: Bearer <token>
```

### 角色说明

| 值 | 角色 | 说明 |
|----|------|------|
| 1 | 学员 | 可报名考试、查询自己的成绩 |
| 2 | 教练 | 可查看名下学员的考试报名状态（不可审核） |
| 3 | 管理员 | 可审核报名、录入成绩、管理场次 |

### 分页接口通用返回

```json
{
  "state": 20000,
  "data": {
    "records": [...],
    "total": 56,
    "pages": 6,
    "current": 1,
    "size": 10
  }
}
```

## 状态定义

### 考试报名状态（exam_registration.status）

| 值 | 含义 | 说明 |
|----|------|------|
| 0 | 待审核 | 学员已提交，等待管理员审核 |
| 1 | 审核通过 | 管理员已通过，可参加考试（已扣除名额） |
| 2 | 审核不通过 | 管理员拒绝（名额不扣） |
| 3 | 已考试 | 已录入成绩（不管合格与否） |

### 考试场次状态（exam_session.status）

| 值 | 含义 |
|----|------|
| 1 | 报名中 |
| 2 | 已满（名额为 0 时自动标记） |
| 3 | 已结束 |

## 接口列表

### 1. 分页查询考试场次（所有角色）

学员报名前查看可选场次。

```
GET /exam-sessions?page=1&size=10&subject=2&licenseType=C1
```

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| page | int | 否 | 页码，默认 1 |
| size | int | 否 | 每页条数，默认 10 |
| subject | int | 否 | 科目筛选（1-4） |
| licenseType | string | 否 | 车型筛选（C1/C2/B1/N1...） |

**响应示例：**

```json
{
  "state": 20000,
  "data": {
    "records": [
      {
        "id": 1,
        "subject": 2,
        "licenseType": "C1",
        "examDate": "2026-06-15",
        "startTime": "09:00:00",
        "location": "南坪科目二考场",
        "venueId": 1,
        "totalQuota": 80,
        "remainingQuota": 79,
        "status": 1,
        "version": 1
      }
    ],
    "total": 5,
    "pages": 1,
    "current": 1,
    "size": 10
  }
}
```

> ⚠️ 前端展示建议：仅显示 `remainingQuota > 0` 且 `status = 1` 的场次。

### 3. 学员报名考试（学员角色）

```
POST /exam-registrations?studentId=4&sessionId=1
```

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| studentId | int | 是 | 学员 ID（必须等于当前登录用户 ID） |
| sessionId | int | 是 | 考试场次 ID |

**自动校验规则（按顺序）：**

| 校验项 | 不通过提示 |
|--------|-----------|
| 只能为自己报名 | "只能为自己报名考试" |
| 场次是否存在 | 404 |
| 名额是否充足 | "该场次名额已满" |
| 学员车型 vs 场次车型是否匹配 | "该场次仅限 XX 车型报名，您的车型为 YY" |
| 学时是否达标 | "学时不足，当前 X 小时，需 Y 小时" |

**成功响应：**
```json
{
  "state": 20000,
  "data": null
}
```

**失败响应示例（名额不足）：**
```json
{
  "state": 40900,
  "message": "该场次名额已满",
  "data": null
}
```

> ⚠️ 注意：`studentId` 必须等于当前登录用户的 ID，否则会返回 403。前端无需让用户选择自己的 ID，直接传当前用户 ID 即可。

### 5. 审核考试报名（管理员角色）

```
PUT /exam-registrations/{id}/audit?pass=true
```

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | int | 路径参数 | 报名记录 ID |
| pass | boolean | 是 | true=通过，false=不通过 |

**审核通过时的系统行为：**
1. 扣减场次剩余名额（`remainingQuota - 1`）
2. 若名额归零，场次状态自动变为"已满"
3. 生成《准考证》PDF，存入文件记录
4. 报名状态变为"审核通过"

**审核不通过时的系统行为：**
1. 报名状态变为"审核不通过"
2. 名额不扣减

**成功响应：**
```json
{
  "state": 20000,
  "data": null
}
```

**冲突响应（名额已被其他管理员占用）：**
```json
{
  "state": 40900,
  "message": "名额已被其他管理员占用，请刷新后重试",
  "data": null
}
```

> ⚠️ 前端遇到此冲突提示时，建议刷新列表让管理员重新查看。

---

### 6. 录入考试成绩（管理员角色）

```
PUT /exam-registrations/{id}/score?score=95
```

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | int | 路径参数 | 报名记录 ID |
| score | int | 是 | 0-100 的整数成绩 |

**系统行为：**
1. 成绩从系统配置表读取合格分数线（默认 90 分）
2. `score >= 分数线` → 合格（`passStatus = 1`）
3. `score < 分数线` → 不合格（`passStatus = 0`），补考次数自动 +1
4. 报名状态变为"已考试"

**成功响应：**
```json
{
  "state": 20000,
  "data": null
}
```

### 8. 查询我的各科成绩（学员角色）

类似于上一个接口，但是返回更友好的格式，附带场次日期和地点。

```
GET /exam-registrations/my-scores/{studentId}
```

**响应示例：**
```json
{
  "state": 20000,
  "data": [
    {
      "subject": 1,
      "subjectName": "科目一",
      "score": 95,
      "passStatus": 1,
      "passResult": "合格",
      "status": 3,
      "statusDesc": "已考试",
      "retakeCount": 0,
      "applyTime": "2026-05-25T10:00:00",
      "examDate": "2026-06-10",
      "location": "南岸区车管所"
    },
    {
      "subject": 2,
      "subjectName": "科目二",
      "score": null,
      "passStatus": null,
      "passResult": "待考试",
      "status": 1,
      "statusDesc": "审核通过",
      "retakeCount": 0,
      "applyTime": "2026-06-01T10:00:00",
      "examDate": "2026-06-15",
      "location": "南坪科目二考场"
    }
  ]
}
```

**passResult 取值：**

| 值 | 说明 |
|----|------|
| "待考试" | 还没出成绩 |
| "合格" | 已通过 |
| "不合格" | 未通过 |

**statusDesc 取值：**

| 值 | 说明 |
|----|------|
| "待审核" | 管理员还没审 |
| "审核通过" | 可以参加考试 |
| "审核不通过" | 报名被拒 |
| "已考试" | 成绩已出 |

### 9. 教练查看名下学员的考试报名（教练角色）

```
GET /coach-portal/exam-registrations
```

无需额外参数，系统通过 JWT Token 自动识别当前教练身份。

**响应示例：**
```json
{
  "state": 20000,
  "data": [
    {
      "id": 1,
      "studentId": 4,
      "studentName": "王小明",
      "sessionId": 1,
      "subject": 2,
      "status": 0,
      "statusDesc": "待审核",
      "score": null,
      "passStatus": null,
      "retakeCount": 0,
      "applyTime": "2026-06-01T10:00:00",
      "auditTime": null,
      "examDate": "2026-06-15",
      "startTime": "09:00:00",
      "location": "南坪科目二考场",
      "licenseType": "C1"
    }
  ]
}
```

> ⚠️ 教练只能查看，没有任何审核/操作按钮。前端展示时，操作列可以隐藏或置灰。

### 10. 下载准考证（已登录用户）

审核通过时会自动生成准考证 PDF，可通过文件接口下载：

```
GET /files/user/{userId}
```

返回该登录用户的所有文件（含报名表和准考证）：

```json
{
  "state": 20000,
  "data": [
    {
      "id": 1,
      "userId": 4,
      "fileName": "准考证_王小明_科目2.pdf",
      "filePath": "admission_ticket/4_xxx_准考证_王小明_科目2.pdf",
      "fileType": "admission_ticket",
      "uploadTime": "2026-06-02T12:00:00"
    }
  ]
}
```

下载某个文件：
```
GET /files/{id}/download
```

返回文件二进制流，浏览器可触发下载。

前端可筛选 `fileType = "admission_ticket"` 只展示准考证，`fileType = "registration_pdf"` 只展示报名表。

## 前端建议的页面与交互

### 学员端

| 页面 | 调用的接口 | 说明 |
|------|-----------|------|
| 考试报名页 | `GET /exam-sessions` → `POST /exam-registrations` | 选场次 → 确认报名 |
| 我的考试页 | `GET /exam-registrations/my-scores/{id}` | 展示各科状态和成绩 |
| 文件中心 | `GET /files/user/{id}` | 下载准考证/报名表 |

### 管理端

| 页面 | 调用的接口 | 说明 |
|------|-----------|------|
| 报名审核列表 | `GET /exam-registrations` | 分页查看所有报名 |
| 审核操作 | `PUT /exam-registrations/{id}/audit?pass=true/false` | 通过/拒绝 |
| 成绩录入 | `PUT /exam-registrations/{id}/score?score=XX` | 输入成绩提交 |

### 教练端

| 页面 | 调用的接口 | 说明 |
|------|-----------|------|
| 学员考试状态 | `GET /coach-portal/exam-registrations` | 只看不操作 |
