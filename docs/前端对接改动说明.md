# 排班管理与车辆管理 — 前端对接改动说明

> 基于 `master` 分支，本次改动涉及：**新增**排班+车辆管理模块、**修复**约课历史 Bug、**增强**合场响应。

---

## 一、新增模块

### 1.1 车辆管理（管理员专用）

**基础路径**: `/vehicles`

| 方法 | 路径 | 用途 | 请求方式 |
|------|------|------|---------|
| `GET` | `/vehicles` | 分页查询车辆列表 | Query: `page, size, vehicleType(可选), status(可选)` |
| `GET` | `/vehicles/available` | 查可用车辆（教练也可调用） | Query: `vehicleType(可选)` |
| `GET` | `/vehicles/{id}` | 查询车辆详情 | Path: `id` |
| `POST` | `/vehicles` | 新增车辆 | Body: `{ plateNumber, vehicleType, brand?, model?, seats?, status?, remarks? }` |
| `PUT` | `/vehicles/{id}` | 修改车辆 | Body: 同新增，只传需修改字段 |
| `DELETE` | `/vehicles/{id}` | 删除车辆（逻辑删除） | — |

**车辆实体字段**：`id, plateNumber, vehicleType, brand, model, seats, status(1空闲/2使用中/3维修/4报废), remarks`

---

### 1.2 排班管理（三端）

**排班状态机**：`0待审核 → 1已通过(管理员审批) → 3已完成`；`0/1 → 4已取消`；`0 → 2已拒绝`

#### 教练端 (`/coach-portal/schedules`)

| 方法 | 路径 | 用途 |
|------|------|------|
| `POST` | `/coach-portal/schedules` | 提交排班申请 |
| `GET` | `/coach-portal/schedules` | 查看我的排班（支持 `?status=` 筛选） |
| `GET` | `/coach-portal/schedules/pending` | 查看我待审核的排班 |
| `PUT` | `/coach-portal/schedules/{id}/cancel` | 取消排班 |

**提交排班申请请求体**：
```json
{
  "vehicleId": 1,
  "venueId": 4,
  "licenseType": "C1",
  "startTime": "2026-06-10T09:00:00",
  "endTime": "2026-06-10T11:00:00",
  "maxStudents": 1,
  "applyReason": "周三上午常规教学"
}
```

> **前端流程**：教练进入排班页 → 调 `GET /vehicles/available` 取可选车辆 → 调 `GET /venues?venueType=2` 取训练场地 → 填表单提交

#### 管理员端 (`/schedules`)

| 方法 | 路径 | 用途 |
|------|------|------|
| `GET` | `/schedules` | 查所有排班（支持 `coachId, vehicleId, status, startDate, endDate` 筛选，分页） |
| `GET` | `/schedules/pending` | 查待审核排班列表 |
| `GET` | `/schedules/{id}` | 查排班详情 |
| `PUT` | `/schedules/{id}/audit` | 审核排班 |

**审核排班参数**（form-urlencoded）：`status=1(通过) 或 2(拒绝)`，可选 `remark=备注`

#### 学员端 (`/schedules/available`)

| 方法 | 路径 | 用途 |
|------|------|------|
| `GET` | `/schedules/available` | 查看我教练的可约时段 |

> 可选 `?licenseType=C1` 筛选车型。自动查学员绑定的教练，返回该教练 `status=1` 且 `bookedCount < maxStudents` 且时间在未来的排班。**前置条件**：学员必须已绑定教练。

## 二、约课模块改动

### 2.1 状态定义变更

| status | 旧含义 | 新含义 | 说明 |
|--------|-------|--------|------|
| 2 | 文档写"已完成" | **已拒绝** | 修正：2 一直是 REJECTED，文档有误 |
| 4 | 不存在 | **已完成** | **新增**状态，教练确认上课后标记 |

前端建议展示：
```
0=待确认(橙)  1=已确认(蓝)  2=已拒绝(红)  3=已取消(灰)  4=已完成(绿)
```

### 2.2 新增/修改的接口

| 方法 | 路径 | 变更类型 | 说明 |
|------|------|---------|------|
| `GET` | `/appointments/my` | **新增** | 学员查自己约课，分页，支持 `?status=` 筛选 |
| `POST` | `/appointments` | **变更** | 请求体**不再传 `studentId`**，后端从 Token 取；新增 `scheduleId` 字段关联排班 |
| `PUT` | `/appointments/{id}/cancel` | **变更** | 只可取消 0/1 状态；取消后自动释放排班名额 |
| `GET` | `/appointments/{id}` | **变更** | 返回含姓名的详情 |
| `GET` | `/appointments` | **变更** | 现在**限管理员**调用（之前任意登录用户可查全部） |
| `PUT` | `/appointments/{id}/admin-cancel` | **新增** | 管理员取消任意约课 |
| `PUT` | `/coach-portal/appointments/{id}/complete` | **新增** | 教练标记约课完成（status 1→4） |

### 2.3 学员约课请求体变更

**旧格式**（前端之前可能传的）：
```json
{
  "studentId": 4,
  "coachId": 1,
  "startTime": "2026-06-10T09:00:00",
  "endTime": "2026-06-10T10:00:00"
}
```

**新格式**（去掉 `studentId`，新增 `scheduleId`）：
```json
{
  "coachId": 1,
  "scheduleId": 5,
  "startTime": "2026-06-10T09:00:00",
  "endTime": "2026-06-10T10:00:00"
}
```

> `scheduleId` 建议必传。传了之后后端会校验排班归属+时段范围+剩余名额+自动扣减名额。不传则只校验教练绑定关系。

### 2.4 约课接口返回格式变更

**之前**：所有约课接口返回 `Appointment` 实体，只有 `studentId`、`coachId`（整数），**无姓名**。

**现在**：所有约课接口返回含姓名的 Map：

```json
{
  "id": 1,
  "studentId": 4,
  "coachId": 1,
  "scheduleId": 5,
  "startTime": "2026-06-10T09:00:00",
  "endTime": "2026-06-10T10:00:00",
  "status": 0,
  "cancelReason": null,
  "createTime": "2026-06-05T12:00:00",
  "updateTime": null,
  "studentName": "王小明",
  "studentPhone": "15912340001",
  "studentLicenseType": "C1",
  "coachName": "张教练"
}
```

> 涉及接口：`GET /appointments/my`、`GET /appointments`、`GET /appointments/{id}`、`POST /appointments`、`PUT /appointments/{id}/cancel`、`PUT /appointments/{id}/admin-cancel`、`PUT /coach-portal/appointments/{id}/confirm`、`PUT /coach-portal/appointments/{id}/reject`、`PUT /coach-portal/appointments/{id}/complete`、`GET /coach-portal/appointments/pending`、`GET /coach-portal/appointments/calendar`

## 三、合场模块改动

### 3.1 返回格式变更

所有合场接口现在返回含 `coachName` 和 `studentName` 的详情 Map（之前单个操作返回原始实体，无姓名）。

涉及接口：`POST /familiarizations/apply`、`PUT /familiarizations/{id}/pay`、`PUT /familiarizations/{id}/schedule`、`PUT /familiarizations/{id}/complete`、`PUT /familiarizations/{id}/cancel`

> `GET /familiarizations/my` 和 `GET /familiarizations` 之前已经含姓名，无需改动。

---

## 四、教练空闲时间改动

`GET /coach-portal/available-time` 返回结构变更：

```json
{
  "manual": { "monday": ["09:00-12:00"] },
  "schedules": [
    {
      "scheduleId": 5,
      "vehicleId": 1,
      "venueId": 4,
      "licenseType": "C1",
      "startTime": "2026-06-10T09:00:00",
      "endTime": "2026-06-10T11:00:00",
      "maxStudents": 2,
      "bookedCount": 1,
      "remaining": 1
    }
  ]
}
```

> `manual` 是旧的自由 JSON 设置（保留），`schedules` 是从已批准排班自动派生的可约时段，**优先展示 schedules**。

## 五、权限改动

| 接口 | 旧权限 | 新权限 | 原因 |
|------|--------|--------|------|
| `GET /vehicles/available` | 管理员 | **管理员+教练** | 教练排班需选车辆 |
| `GET /venues` | 管理员 | **管理员+教练** | 教练排班需选场地 |
| `GET /venues/{id}` | 管理员 | **管理员+教练** | 同上 |
| `GET /appointments` | 任意登录用户 | **管理员** | 不应泄露全部约课数据 |

---

## 六、前端需要调整的清单

1. **车辆管理页（管理员）**：新增页面，调用 `/vehicles` 系列接口，CRUD 车队
2. **排班审核页（管理员）**：新增页面，调用 `GET /schedules/pending` → `PUT /schedules/{id}/audit`
3. **排班申请页（教练）**：新增入口，先调 `/vehicles/available` + `/venues?venueType=2`，再提交 `/coach-portal/schedules`
4. **教练我的排班页（教练）**：新增，调 `/coach-portal/schedules` 查看/取消
5. **学员约课页**：
   - 约课前先调 `GET /schedules/available` 获取教练可约时段
   - `POST /appointments` 请求体去掉 `studentId`，加上 `scheduleId`
   - 约课列表改用 `GET /appointments/my`
6. **约课相关页面（全局）**：
   - 所有约课展示改为读取 `studentName`/`coachName` 而非仅 `studentId`/`coachId`
   - 状态新增 `4=已完成`，状态 `2` 的含义从"已完成"改为"已拒绝"
7. **教练约课操作**：完成约课改用 `PUT /coach-portal/appointments/{id}/complete`
8. **合场页面**：apply/pay/schedule/complete/cancel 返回改为包含 `coachName`/`studentName`
9. **教练空闲时间页**：`GET /coach-portal/available-time` 返回新增 `schedules` 字段，优先展示
10. **权限调整**：学员调用 `GET /appointments` 改为 `GET /appointments/my`
