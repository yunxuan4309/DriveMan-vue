# 考试场次安排接口文档

## 通用说明

### 统一响应格式

```json
{
  "state": 20000,
  "message": null,
  "data": { ... }
}
```

| state | 含义 |
|-------|------|
| 20000 | 成功 |
| 40000 | 请求参数错误 |
| 40100 | 未登录或 token 过期 |
| 40300 | 权限不足 |
| 40400 | 资源不存在 |
| 40900 | 数据冲突 |

### 认证方式

所有接口需在请求头中携带 Token：

```
Authorization: Bearer <token>
```

### 角色权限与字段说明

**角色权限：**

| 角色 | 查看场次 | 发布/修改/删除 |
|------|----------|---------------|
| 学员 (role=1) | ✅ | ❌ |
| 教练 (role=2) | ✅ | ❌ |
| 管理员 (role=3) | ✅ | ✅ |

**场次状态：**

| status | 含义 |
|--------|------|
| 1 | 报名中（有剩余名额） |
| 2 | 已满（剩余名额为 0） |
| 3 | 已结束 |

### 分页通用返回

```json
{
  "state": 20000,
  "data": {
    "records": [...],
    "total": 56,
    "pages": 6,
    "current": 1,
    "size": 10
  }
}
```

## 1. 分页查询考试场次

所有登录用户可调用，按考试日期升序排列，可按科目或车型筛选。

```
GET /exam-sessions?page=1&size=10&subject=2&licenseType=C1
```

### 请求参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| page | int | 否 | 页码，默认 1 |
| size | int | 否 | 每页条数，默认 10 |
| subject | int | 否 | 科目筛选（1-4） |
| licenseType | String | 否 | 车型筛选（C1/C2/B1/N1 等） |

### 响应示例

```json
{
  "state": 20000,
  "data": {
    "records": [
      {
        "id": 1,
        "subject": 1,
        "licenseType": "C1",
        "examDate": "2026-06-10",
        "startTime": "09:00:00",
        "location": "南岸区车管所",
        "venueId": 1,
        "totalQuota": 100,
        "remainingQuota": 98,
        "status": 1,
        "version": 1,
        "createTime": "2026-06-01T10:00:00"
      }
    ],
    "total": 3,
    "pages": 1,
    "current": 1,
    "size": 10
  }
}
```

### 返回字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| id | int | 场次ID |
| subject | int | 科目（1-4） |
| licenseType | String | 适用车型 |
| examDate | date | 考试日期 |
| startTime | time | 开始时间 |
| location | String | 考试地点 |
| venueId | int | 关联考场ID（可为 null） |
| totalQuota | int | 总名额 |
| remainingQuota | int | 剩余名额 |
| status | int | 场次状态（1-报名中, 2-已满, 3-已结束） |
| version | int | 乐观锁版本号 |

## 3. 发布考试场次

仅管理员可调用。

```
POST /exam-sessions
Content-Type: application/json
```

### 请求体

```json
{
  "subject": 2,
  "licenseType": "C1",
  "examDate": "2026-06-15",
  "startTime": "09:00",
  "location": "南岸区车管所",
  "venueId": 1,
  "totalQuota": 50,
  "remainingQuota": 50,
  "status": 1
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| subject | int | 是 | 科目（1-4） |
| licenseType | String | 是 | 适用车型（C1/C2/B1/N1 等） |
| examDate | String | 是 | 考试日期，格式 yyyy-MM-dd，不能早于当前日期 |
| startTime | String | 否 | 开始时间，格式 HH:mm |
| location | String | 是 | 考试地点 |
| venueId | int | 否 | 关联考场ID |
| totalQuota | int | 是 | 总名额，必须 > 0 |
| remainingQuota | int | 是 | 剩余名额，≤ totalQuota |
| status | int | 否 | 默认 1（报名中） |

### 校验规则

- `subject` 必须在 1-4 范围内
- `examDate` 不能为空，且不能早于当前日期
- `location`、`licenseType` 不能为空
- `totalQuota` 必须大于 0
- `remainingQuota` 不能为负数，且不能超过 `totalQuota`

### 成功响应

```json
{
  "state": 20000,
  "message": null,
  "data": null
}
```

## 4. 修改考试场次

仅管理员可调用，不存在时返回 404。

```
PUT /exam-sessions/{id}
Content-Type: application/json
```

### 请求体

```json
{
  "examDate": "2026-06-16",
  "startTime": "08:30",
  "location": "南坪科目二考场",
  "totalQuota": 80,
  "remainingQuota": 80,
  "status": 1
}
```

修改时只需传入需要变更的字段，其余字段保持原值。校验规则与发布相同（但不校验日期是否早于当前日期）。

### 成功响应

```json
{
  "state": 20000,
  "data": null
}
```

### 错误响应（场次不存在）

```json
{
  "state": 40400,
  "message": "考试场次不存在",
  "data": null
}
```

---

## 5. 删除考试场次

仅管理员可调用，逻辑删除（软删除），不存在时返回 404。

```
DELETE /exam-sessions/{id}
```

### 成功响应

```json
{
  "state": 20000,
  "data": null
}
```

## 接口路径速查

| 操作 | 方法 | 路径 | 角色 |
|------|------|------|------|
| 分页查询场次 | GET | `/exam-sessions` | 全部（登录） |
| 查询场次详情 | GET | `/exam-sessions/{id}` | 全部（登录） |
| 发布场次 | POST | `/exam-sessions` | 管理员 |
| 修改场次 | PUT | `/exam-sessions/{id}` | 管理员 |
| 删除场次 | DELETE | `/exam-sessions/{id}` | 管理员 |

## 典型使用场景

### 学员
1. 调用 `GET /exam-sessions` 查看可选场次和剩余名额
2. 选择场次后调用 `POST /exam-registrations` 报名

### 教练
1. 调用 `GET /exam-sessions` 查看名下学员可报名的场次信息
2. 调用 `GET /exam-sessions/{id}` 查看某个场次的详细信息
3. 在教练工作台查看名下学员的考试报名记录

### 管理员
1. 调用 `POST /exam-sessions` 发布新的考试场次
2. 调用 `PUT /exam-sessions/{id}` 调整场次名额或信息
3. 调用 `DELETE /exam-sessions/{id}` 删除已结束或错误的场次
4. 在考试报名管理中审核学员的报名请求
