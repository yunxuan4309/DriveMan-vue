# 教练与学员管理接口文档

## 通用说明

### 统一响应格式

```json
{
  "state": 20000,       // 业务状态码
  "message": null,      // 错误时携带提示信息
  "data": { ... }       // 成功时携带数据
}
```

| state | 含义 |
|-------|------|
| 20000 | 成功 |
| 40000 | 请求参数错误 |
| 40100 | 未登录或 token 过期 |
| 40300 | 权限不足 |
| 40400 | 资源不存在 |
| 40900 | 数据冲突（如用户名已存在） |

### 认证方式

所有接口（除登录和注册外）需要在请求头中携带 Token：

```
Authorization: Bearer <token>
```

### 角色说明

| 值 | 角色 |
|----|------|
| 1 | 学员 |
| 2 | 教练 |
| 3 | 管理员 |

### 分页接口通用返回

分页接口返回 `Page<T>` 对象：

```json
{
  "state": 20000,
  "data": {
    "records": [...],   // 当前页数据
    "total": 56,        // 总记录数
    "pages": 6,         // 总页数
    "current": 1,       // 当前页码
    "size": 10          // 每页条数
  }
}
```

## 一、登录与注册

### 1.1 学员注册

公开接口，无需登录。

```
POST /register
Content-Type: application/json

{
  "username": "student3",
  "password": "admin123",
  "realName": "张三",
  "idCard": "500101200205036789",
  "phone": "15912340003",
  "address": "重庆市南岸区",
  "licenseType": "C1"
}
```

**响应：**
```json
{
  "state": 20000,
  "data": { "userId": 6 }
}
```

> 注册后状态为 `status=0`（待审核），需管理员审核通过才能登录。

### 1.2 登录

```
POST /login
Content-Type: application/x-www-form-urlencoded

username=student1&password=admin123
```

**响应：**
```json
{
  "state": 20000,
  "data": {
    "token": "eyJhbGciOiJIUzI1NiJ9...",
    "userId": 4,
    "username": "student1",
    "realName": "王小明",
    "role": 1
  }
}
```

## 二、用户管理（学员/教练/管理员 CRUD）

基础路径：`/users`

### 2.1 分页查询用户

```
GET /users?page=1&size=10&role=1
```

| 参数 | 说明 | 必填 |
|------|------|------|
| page | 页码，默认 1 | 否 |
| size | 每页条数，默认 10 | 否 |
| role | 角色筛选：1-学员，2-教练，3-管理员 | 否 |

> 仅管理员可调用（`@RequireRole(3)`）

**响应：**
```json
{
  "state": 20000,
  "data": {
    "records": [
      {
        "userId": 4,
        "role": 1,
        "username": "student1",
        "realName": "王小明",
        "idCard": "500101200001011234",
        "phone": "15912340001",
        "address": "重庆市南岸区学府大道",
        "licenseType": "C1",
        "avatar": null,
        "status": 1,
        "auditReason": null,
        "createTime": "2026-05-22T12:00:00",
        "updateTime": "2026-05-22T12:00:00"
      }
    ],
    "total": 5,
    "pages": 1,
    "current": 1,
    "size": 10
  }
}
```

### 2.2 查询用户详情

```
GET /users/{userId}
```

> 仅管理员可调用

### 2.3 获取当前登录用户信息

```
GET /users/me
```

> 任意登录用户可调用（从 Token 解析当前用户）

### 2.4 新增用户

```
POST /users
Content-Type: application/json

{
  "username": "student3",
  "password": "admin123",
  "realName": "张三",
  "role": 1,
  "idCard": "500101200205036789",
  "phone": "15912340003",
  "address": "重庆市南岸区",
  "licenseType": "C1"
}
```

> - 仅管理员可调用
> - 密码会自动 BCrypt 加密后入库
> - `status` 默认为待审核

### 2.5 修改用户

```
PUT /users/{userId}
Content-Type: application/json

{
  "realName": "张三改",
  "phone": "15912340099",
  "address": "重庆市江北区"
}
```

> - 仅管理员可调用
> - 不需要修改的字段不传即可
> - `password` 传空字符串或 null 不会覆盖原密码

### 2.6 删除用户（逻辑删除）

```
DELETE /users/{userId}
```

> 仅管理员可调用，数据软删除（`is_deleted=1`），不会真的从数据库移除

### 2.7 修改密码

```
PUT /users/{userId}/password
Content-Type: application/x-www-form-urlencoded

oldPassword=admin123&newPassword=newpass456
```

> 任意登录用户可调用，`oldPassword` 验证不通过则拒绝修改

## 三、教练管理

基础路径：`/coaches`

### 3.1 分页查询教练

```
GET /coaches?page=1&size=10
```

> 任意登录用户可调用

### 3.2 查询教练详情

```
GET /coaches/{coachId}
```

### 3.3 新增教练

```
POST /coaches
Content-Type: application/json

{
  "userId": 2,
  "rating": 4.8,
  "coachYears": 6,
  "vehicleType": "C1,C2,B1"
}
```

> 仅管理员可调用。需要注意：
> - `userId` 必须先创建对应的 User（角色=2）
> - `vehicleType` 以逗号分隔（如 `"C1,C2"`），用于教练推荐匹配

### 3.4 修改教练

```
PUT /coaches/{coachId}
Content-Type: application/json

{
  "rating": 4.9,
  "coachYears": 7,
  "vehicleType": "C1,C2,B1"
}
```

> 仅管理员可调用

### 3.5 删除教练

```
DELETE /coaches/{coachId}
```

> 仅管理员可调用，逻辑删除

## 四、学员报名审核

基础路径：`/registrations`

### 4.1 查询待审核学员

```
GET /registrations/pending
```

> 仅管理员可调用，返回所有 `status=0` 的学员

### 4.2 审核学员报名

```
PUT /registrations/{userId}/audit?pass=true
```

| 参数 | 说明 | 必填 |
|------|------|------|
| pass | true=通过，false=不通过 | 是 |
| reason | 不通过原因（pass=false 时必填） | 否 |

> - 仅管理员可调用
> - 审核通过时：`user.status = 1`，自动生成报名表 PDF + 准考证 PDF
> - 审核不通过时：填写原因，`user.status = 2`

## 五、教练申请与分配

### 5.1 学员申请教练

```
POST /coach-applications
Content-Type: application/x-www-form-urlencoded

studentId=4&coachId=1
```

> 仅学员可调用，提交后等待管理员审核

### 5.2 查询待审核的教练申请

```
GET /coach-applications/pending
```

> 仅管理员可调用

### 5.3 查询学员的申请记录

```
GET /coach-applications/student/{studentId}
```

> 任意登录用户可调用

### 5.4 审核教练申请

```
PUT /coach-applications/{id}/audit?pass=true
```

| 参数 | 说明 | 必填 |
|------|------|------|
| pass | true=通过，false=拒绝 | 是 |
| reason | 拒绝原因（pass=false 时选填） | 否 |

> - 仅管理员可调用
> - 审核通过时会自动写入 `student_coach` 绑定关系

### 5.5 自动推荐教练

```
GET /coach-assignments/recommend?studentId=4&topN=5
```

> 任意登录用户可调用
> 推荐算法：根据学员的 `licenseType` 匹配教练的 `vehicleType` → 按评分降序

### 5.6 手动分配教练

```
POST /coach-assignments
Content-Type: application/x-www-form-urlencoded

studentId=4&coachId=2
```

> 仅管理员可调用。若学员已绑定教练则会报错（需先解绑）

### 5.7 查询所有绑定关系

```
GET /coach-assignments
```

> 仅管理员可调用。返回包括学员姓名和教练姓名：

```json
{
  "state": 20000,
  "data": [
    {
      "id": 1,
      "studentId": 4,
      "coachId": 1,
      "bindTime": "2026-05-22T12:00:00",
      "studentName": "王小明",
      "coachName": "张教练"
    }
  ]
}
```

### 5.8 解绑教练

```
PUT /coach-assignments/{id}/unbind
```

> 仅管理员可调用，将 `student_coach.status` 置为 0（已解绑）

## 六、学员进度查询

基础路径：`/students`

### 6.1 查询学员进度

```
GET /students/{studentId}/progress
```

> 任意登录用户可调用

**响应：**
```json
{
  "state": 20000,
  "data": {
    "studentId": 4,
    "realName": "王小明",
    "licenseType": "C1",
    "progressPercent": 25,
    "subjects": [
      {
        "subject": 1,
        "status": "passed",
        "score": 95,
        "description": "科目一 道路交通安全法律、法规和相关知识",
        "examItems": null,
        "requiredHours": 0,
        "trainedHours": 0,
        "remainingHours": 0
      },
      {
        "subject": 2,
        "status": "learning",
        "description": "科目二 场地驾驶技能",
        "examItems": ["倒车入库", "侧方停车", "坡道定点停车和起步", "直角转弯", "曲线行驶"],
        "requiredHours": 16,
        "trainedHours": 14,
        "remainingHours": 2
      },
      {
        "subject": 3,
        "status": "locked",
        "description": "科目三 道路驾驶技能",
        "examItems": ["上车准备", "起步", ...],
        "requiredHours": 24,
        "trainedHours": 0,
        "remainingHours": 24
      },
      {
        "subject": 4,
        "status": "locked",
        "description": "科目四 安全文明驾驶常识",
        "examItems": null,
        "requiredHours": 0,
        "trainedHours": 0,
        "remainingHours": 0
      }
    ]
  }
}
```

**status 含义：**

| 值 | 说明 | 前端显示 |
|----|------|---------|
| `locked` | 前一科未通过，此科锁定 | 灰色锁图标 |
| `learning` | 学习中，学时未达标 | 进度条（如 14/16） |
| `ready` | 学时已达标，可以报名考试 | 绿色"可报名"按钮 |
| `passed` | 考试已通过 | ✅ 绿色打勾 |

**状态流转规则：**
```
科目一 → passed → 科目二 → unlocked → learning → ready → passed → 科目三 → ...
                                    ↓
                              学时不足，继续练
```
